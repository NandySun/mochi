## 阶段八细节：全屏架构变更

### 8.1 焦点指示器

海报横滚下方一排小圆点，当前选中高亮。仅 `filtered.length > 1` 时显示。

### 8.2 原生全屏实验（成功采用）

**实验前**：全屏 = Tauri 原生 `set_fullscreen` + `createPortal` CSS 浮层（双轨，OSC 渲染两份，auto-hide 逻辑分离）。

**实验结果**：原生全屏 + 透明窗口 + mpv 视频透底正常工作。portal 方案被替换。

**最终方案**：统一布局（`VideoPlayer.tsx` 一份 JSX），全屏时通过 `isFullscreen` 条件切换样式：
- 视频区 `margin:0; borderRadius:0` 撑满窗口
- 顶栏/OSB 3s 无操作自动淡出（`opacity 0 + pointer-events:none`），鼠标移动重现
- 光标 3s 后变 `none`
- 全屏按钮图标动态切换（向内箭头→向外箭头）

**删除**：整个 portal 相关代码（`createPortal`、`fullscreenShowCtrls`、`fsTimerRef`、`resetFullscreenTimer`），代码量净减少约 60 行。

## 当前架构

| 路由 | 组件 | 职责 |
|---|---|---|
| `/` | PosterWall | 海报横滚 + 筛选栏（全部/继续/动漫/影视） + 标题 + 焦点圆点 + 设置入口 |
| `/series/:id` | SeriesDetail | 系列信息 + 剧集列表（纯浏览） |
| `/play/:episodeId` | VideoPlayer | 纯播放：顶栏 + 视频区 + OSC（含选集📋），全屏时 auto-hide |
| 弹窗 | Settings | 媒体库/元数据/关于，portal 注入 body |

全屏：Tauri 原生 `set_fullscreen` + 统一布局条件样式（无 portal，视频 margin/borderRadius 置零，OSC+光标 3s 淡出）。
