> 2026-06-29 起草 · 基于 v0.3.0 搜索/快捷键里程碑之后

## 一、概述

为 mochi 新增独立统计页面 `/stats`，展示个人观影数据。设计目标：

- **手绘感冲淡黑金严肃**：折线图用厚涂笔触模拟记号笔手绘，hero 数字用 Pristina 手写体
- **一屏不滚动**：所有信息在 16:9 窗口内完成，hero 居中悬浮留白区域，时间线沉底
- **数据先行**：新增 `watch_events` 表，从 v0.3.1 起开始积累时间序列数据，不做历史回溯

## 二、数据层

### 2.1 watch_events 表

```sql
CREATE TABLE IF NOT EXISTS watch_events (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    episode_id    INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
    series_id     INTEGER NOT NULL REFERENCES series(id) ON DELETE CASCADE,
    progress_secs INTEGER NOT NULL,
    timestamp     TEXT    NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX idx_watch_events_ts     ON watch_events(timestamp);
CREATE INDEX idx_watch_events_series ON watch_events(series_id, timestamp);
```

### 2.2 写入逻辑

`update_watch_progress` 每次调用时（每 10s + 切集 + 关闭前），追加一条记录。不改变现有 progress 更新逻辑，仅多一行 INSERT。

### 2.3 聚合查询

**每日观看时长（过去 30 天）**：

```sql
SELECT date(timestamp) as day,
       SUM(daily_max - daily_min) as watch_secs
FROM (
  SELECT episode_id, date(timestamp) as day,
         MAX(progress_secs) as daily_max,
         MIN(progress_secs) as daily_min
  FROM watch_events
  WHERE timestamp >= datetime('now', '-30 days')
  GROUP BY episode_id, date(timestamp)
)
GROUP BY day
ORDER BY day
```

取每集每天的 MAX - MIN 作为有效观看时长，避免重复计数。

**总观看时长**：`SUM(daily_max - daily_min)` 全局聚合。

**最近观看**：

```sql
SELECT DISTINCT episode_id, series_id, MAX(timestamp) as last_ts
FROM watch_events
GROUP BY episode_id
ORDER BY last_ts DESC
LIMIT 8
```

### 3.3 设计要点

| 元素 | 规格 |
|---|---|
| Hero 数字 | Pristina 手写体 · 约 110pt · `#c47e3a` · 居中悬浮 · 上下留白 ≥ 15% 页面高度 |
| Hero 单位/副标题 | Pristina · 约 40pt · `#8a5a2e` / Segoe UI 12pt · `#333` |
| 折线图线体 | 粗 4-5px · 5 层笔触叠加 · Catmull-Rom 样条 · roughness 0.9 |
| 坐标轴 | 粗 1.5-2px · roughness 0.5 · `rgba(255,255,255,0.1)` |
| 数据点 | 平时不可见 · hover 浮现 4px 圆点 + tooltip |
| 时间线条目 | 4 条 · 海报 48×72 · 行间距 ≥ 60px · 单列或宽松双列 |
| 海报占位 | 渐变色块 + 首字母 + rough.js 手绘边框 |
| 背景 | `#0b0b0b` 纯黑 · 无 fanart · 无 gradient |

### 3.4 折线图手绘策略

**厚线条**：5 层 SVG path 叠加，主路径 0.7 透明度 + 4 层副路径各 0.12-0.18。每层独立伪随机偏移。视觉等效于记号笔在纸上反复描摹一道线——边缘模糊、粗细不匀、笔触重叠。

**坐标轴加粗**：横轴和纵轴线条同样加粗到 1.5-2px，roughness 0.5，与折线形成一致的"硬笔手绘"质感。标签数字用 Segoe UI 10pt，不做手绘抖动。

**纵向充裕**：图表高度 ≥ 250px，给折线的起伏足够的纵向展开空间。横向收窄到约 60% 页面宽度，居中放置。

**hover 交互**：平时只有线。16px 半径透明热区分布在数据点位置，hover 时浮现 4px 金色圆点 + tooltip（日期 + 分钟数）。

## 六、前置决策

1. **最近观看条目数**：4 条，双列。减少条目是为了放大海报和文字，不是为留白
2. **hero 副标题**：动态数据（如"自 2025 年 9 月起"）。比页面其他灰色偏白半级以示区分，字号放大，hero 正下方居中。数字部分同样用 Pristina 手写体
