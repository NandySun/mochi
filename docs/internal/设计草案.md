## 一、核心愿景

一个**桌面原生、封面即导航、多媒介一体的本地影库**：

- 打开应用 → 看到海报墙，每张海报一个系列
- 点击海报 → 进入该系列，看到剧集列表
- 点击剧集 → 内置播放器直接播放
- 未来扩展漫画、音乐，同一入口管理所有个人媒体
- 跨平台：Windows → macOS 可平滑迁移

## 二、设计原则

1. **桌面原生感**：独立窗口应用，自定义标题栏，系统托盘，窗口位置记忆
2. **海报即导航**：封面是唯一入口，不存在侧边栏"文件浏览器"
3. **零配置入库**：指定根目录，递归扫描，自动按文件夹分组。支持四种类型（anime/movie/tv/variety），通过父文件夹名、文件夹后缀 `[type]`、或添加目录时手动指定类型来识别。元数据 API 无法识别的类型（如综艺）由用户显式标记。
4. **按需拉元数据**：动漫走 Bangumi（免费无 Key），影视走 TMDB（免费 Key），离线优先
5. **本地优先**：数据存 SQLite，元数据缓存到磁盘，断网也能用
6. **渐进式扩展**：v0.1 只做视频，漫画和音乐是独立里程碑
7. **双轨分发**：安装版（NSIS）和便携版（zip）从同一代码产出，通过 `.portable` 标记切换数据路径

## 三、技术栈

| 层 | 技术 |
|---|---|
| 框架 | Tauri v2 |
| 前端 | React 19 + TypeScript + Tailwind CSS + framer-motion 11 |
| 数据 | SQLite（rusqlite + bundled） |
| 播放 | libmpv + tauri-plugin-libmpv |
| 元数据 | Bangumi + TMDB 双搜 |

**mpv 嵌入方案**：透明窗口 `transparent: true`，视频帧渲染到原生窗口表面（WebView 之下），WebView 半透明层让视频透出。

## 四、数据模型

```
Series
  ├── title, type(anime/tv/movie/variety/unknown)
  ├── posterPath, fanartPath
  ├── bangumiId / tmdbId
  ├── synopsis, year, score, genres

Episode
  ├── seriesId, seasonNumber, episodeNumber
  ├── filePath, duration, subtitleCount
  ├── status(ready/downloading/missing)
  ├── watchedProgress, watchedCompleted
  ├── title, stillPath, stillUrl           ← 新增：剧集名称 + 缩略图（TMDB）
  ├── overview, airDate, runtime           ← 新增：剧集元数据（可选）

Person                                    ← 新增：演员/声优表
  ├── source(tmdb/bangumi), sourceId
  ├── name, roleName
  ├── imageUrl, imageCache

SeriesCast                                ← 新增：作品-人员关联
  ├── seriesId, personId, sortOrder
```

## 五、UI 架构

### 路由

| 路由 | 页面 | 职责 |

| `/` | PosterWall | 海报横滚 + 筛选栏（全部/继续/动漫/影视/电影/综艺）+ 设置入口 |

### 窗口管理

- `decorations: false` 自定义标题栏，关闭按钮走 `window_close` 命令（非原生 CloseRequested）
- 系统托盘：左键显示窗口，右键菜单（显示窗口 / 退出）
- 窗口关闭行为可配置（设置 → 通用）：最小化到托盘 / 彻底退出
- 托盘右键「退出」始终真正退出，不受关闭行为设置影响
- 两套关闭路径各自处理：标题栏 ✕ → `window_close` 命令，Alt+F4 → `CloseRequested` 事件

### 全局背景层

`GlobalBackground`（App 级渲染，三层：gradient + fanart + mask）：
- `position: fixed; z-index: 0; pointer-events: none`
- 跨路由常驻，由各页面通过 `useBackground().setBg()` 控制
- 浏览页：完整 gradient(rgba 0.4) + fanart(opacity 0.35) + mask
- 播放页：gradient 透明 + fanart 空，视频透底无雾

## 七、元数据双搜策略（v0.2.1 更新）

| 类型 | 主搜 | 横幅来源 | 说明 |
|---|---|---|---|
| anime | Bangumi（海报/简介/评分） | TMDB backdrop | Bangumi 不提供横幅，有 TMDB Key 则补拉 |
| tv / movie / variety | TMDB | TMDB backdrop | Bangumi 兜底。variety 不在 Bangumi 覆盖范围，必须走 TMDB |

- 已有 `bangumi_id` 的系列走 ID 快速通道，获取后在 `commands.rs` 外层统一补 TMDB 横幅
- `build_bangumi_result` 不设 `tmdb_id` 和 `fanart`（防止跨源污染），由外层补拉
- TMDB 横幅搜索用 `effective_term`，匹配失败静默跳过
- Anime 无视已存储的 `tmdb_id`（防止 TMDB 兜底污染后的 ID 快速路径返回错误类型）

### 文件夹季号识别（v0.2.1）

扫描器从文件夹名自动提取季号，支持三种模式：
- 中文：`第X季`（含中文数字「第六季」→ 6）
- 英文：`Season X`
- 简写：`SX`（仅末尾）

提取后：`search_term` 剥离季号用于 API 搜索，`display_name` 保留完整名称，episode 默认 season_number 设为提取值（SXXEYY 等文件名显式季号优先）。

### 季节级系列元数据（v0.2.1）

`fetch_episode_metadata` 检测到所有剧集属于同一季时，从 TMDB season 接口额外拉取：
- 季海报（覆盖系列级 poster）
- 季简介（覆盖系列级 synopsis）
- 季名称（拼入系列 title：「哈哈哈哈哈」+「第六季」→「哈哈哈哈哈 第六季」）

## 八、响应式策略

mochi 是桌面应用，Tauri 窗口最小宽度 780px。三个断点：

| 断点 | 宽度 | 海报 | 演员头像 | 剧集卡片 | 其他 |
|------|------|------|----------|----------|------|
| L | ≥ 1100px | 186×280 | 48px | 180×101 | 完整布局 |
| M | 900–1100px | 160px 宽等比 | 40px | 150×84 | 标题 24px，hero 间距 24px |
| S | 780–900px | 130px 宽等比 | 36px | 130×73 | 标题 20px，声优名隐藏 |

不在任何桌面宽度下堆叠成纵向单列。海报始终在左侧。

## 九、设计语言

mochi 的交互设计遵循**材质物理三定律**：

1. **形变可逆** — 按压、拉伸、挤压都是暂时响应，最终回归原始形态
2. **运动有质量** — 所有运动携带惯性，启动有延迟、停止有回弹
3. **温度是信息** — 颜色和光影变化承载叙事，不纯做装饰

核心交互符号：
- **呼吸的圆** — 唯一加载态，所有等待场景共享同一符号
- **果冻形变** — 筛选指示器 scaleX 拉伸 + overshoot 回弹
- **按压 squish** — `whileTap scale` 覆盖所有可交互表面
- **粘度滚动** — JS 软 snap + 磁吸 + 橡胶带回坐
- **颜色温度叙事** — 系列间背景 crossfade 不对称曲线

## 十、已知限制

- Windows only（macOS 计划中）
- 音轨/字幕切换中 `track-list` 部分功能不可用（libmpv-wrapper 兼容性）
- 不支持自动截帧缩略图，封面优先使用远程拉取的 Bangumi/TMDB 图片
- `api.bgm.tv` 在国内网络环境下不可达（需代理，且源站可能间歇 502）
- Unicode 符号用于菜单项（非 emoji），在极少数系统字体下可能回退显示
