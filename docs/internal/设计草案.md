# Mochi — 设计草案

> 2026-06-16 起草，2026-06-21 更新至设计语言深化完成。

---

## 一、核心愿景

一个**桌面原生、封面即导航、多媒介一体的本地影库**：

- 打开应用 → 看到海报墙，每张海报一个系列
- 点击海报 → 进入该系列，看到剧集列表
- 点击剧集 → 内置播放器直接播放
- 未来扩展漫画、音乐，同一入口管理所有个人媒体
- 跨平台：Windows → macOS 可平滑迁移

## 二、设计原则

1. **桌面原生感**：独立窗口应用，自定义标题栏，系统托盘，窗口位置记忆
2. **海报即导航**：封面是唯一入口，不存在侧边栏"文件浏览器"
3. **零配置入库**：指定根目录，递归扫描，自动按文件夹分组
4. **按需拉元数据**：动漫走 Bangumi（免费无 Key），影视走 TMDB（免费 Key），离线优先
5. **本地优先**：数据存 SQLite，元数据缓存到磁盘，断网也能用
6. **渐进式扩展**：v0.1 只做视频，漫画和音乐是独立里程碑

## 三、技术栈

| 层 | 技术 |
|---|---|
| 框架 | Tauri v2 |
| 前端 | React 19 + TypeScript + Tailwind CSS + framer-motion 11 |
| 数据 | SQLite（rusqlite + bundled） |
| 播放 | libmpv + tauri-plugin-libmpv |
| 元数据 | Bangumi + TMDB 双搜 |

**mpv 嵌入方案**：透明窗口 `transparent: true`，视频帧渲染到原生窗口表面（WebView 之下），WebView 半透明层让视频透出。

## 四、数据模型

```
Series
  ├── title, type(anime/tv/movie/unknown)
  ├── posterPath, fanartPath
  ├── bangumiId / tmdbId
  ├── synopsis, year, score, genres

Episode
  ├── seriesId, seasonNumber, episodeNumber
  ├── filePath, duration, subtitleCount
  ├── status(ready/downloading/missing)
  ├── watchedProgress, watchedCompleted
```

## 五、UI 架构

### 路由

| 路由 | 页面 | 职责 |
|---|---|---|
| `/` | PosterWall | 海报横滚 + 筛选栏（全部/继续/动漫/影视）+ 设置入口 |
| `/series/:id` | SeriesDetail | 系列信息 + 剧集列表，fanart 全屏背景 |
| `/play/:episodeId` | VideoPlayer | 纯播放：顶栏 + 视频区 + OSC（含选集📋）+ 全屏 auto-hide |
| 弹窗 | Settings | 媒体库/元数据/OSC 主题/关于，portal 注入 body |

### 全局背景层

`GlobalBackground`（App 级渲染，三层：gradient + fanart + mask）：
- `position: fixed; z-index: 0; pointer-events: none`
- 跨路由常驻，由各页面通过 `useBackground().setBg()` 控制
- 浏览页：完整 gradient(rgba 0.4) + fanart(opacity 0.35) + mask
- 播放页：gradient 透明 + fanart 空，视频透底无雾

## 六、OSC 主题系统

5 个内置预设，定义在 `src/themes/oscThemes.ts`：

| ID | 名称 | 特征 |
|---|---|---|
| `mochi` | Mochi 默认 | 暖金 accent（#c47e3a），圆形按钮，spread 布局 |
| `youtube` | YouTube | 红 accent（#ff0000），居中播放键，hover 加粗进度条 |
| `potplayer` | PotPlayer | 蓝 accent（#1e88e5），紧凑布局，方形小按钮 |
| `netflix` | Netflix | 白 accent，极简，隐藏倍速/音量独立滑块 |
| `minimal` | 极简 | 灰白 accent，只留播放/进度 |

新增主题只需在 `oscThemes.ts` 中添加一个 `OscTheme` 对象。设置页自动显示。

## 七、设计语言

mochi 的交互设计遵循**材质物理三定律**（详见 `设计语言深化方案.md`）：

1. **形变可逆** — 按压、拉伸、挤压都是暂时响应，最终回归原始形态
2. **运动有质量** — 所有运动携带惯性，启动有延迟、停止有回弹
3. **温度是信息** — 颜色和光影变化承载叙事，不纯做装饰

核心交互符号：
- **呼吸的圆** — 唯一加载态，所有等待场景共享同一符号
- **果冻形变** — 筛选指示器 scaleX 拉伸 + overshoot 回弹
- **按压 squish** — `whileTap scale` 覆盖所有可交互表面
- **粘度滚动** — JS 软 snap + 磁吸 + 橡胶带回坐
- **颜色温度叙事** — 系列间背景 crossfade 不对称曲线

## 八、已知限制

- Windows only（macOS 计划中）
- 音轨/字幕切换中 `track-list` 部分功能不可用（libmpv-wrapper 兼容性）
- 不支持自动截帧缩略图，需手动在每个系列文件夹放置 `poster.jpg`
- 封面图片通过 Tauri 后端读取 base64 返回前端，大量图片时可能有性能瓶颈
