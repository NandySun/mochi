> 2026-06-29 · v0.3.1 发布 · 莫兰迪暖灰 Palette + 对比度增强

## 项目概览

桌面原生个人多媒体库。Tauri v2 + React 19 + TypeScript + Tailwind + SQLite + libmpv + framer-motion 11。
本版本聚焦首页浏览效率：搜索快速定位系列，Indicator Dots 可点击跳转，快捷键统一在设置页展示。
GitHub 公开仓库：`https://github.com/NandySun/mochi`

## 当前架构

| 路由/覆盖层             | 组件           | 职责                                      |
| ------------------ | ------------ | --------------------------------------- |
| `/`                | PosterWall   | 海报横滚 + 筛选栏（全部/继续/动漫/影视/电影/综艺）+ 搜索按钮 + Indicator Dots |
| `/series/:id`      | SeriesDetail | 杂志跨页：海报 + 元数据/演员双区 + 剧集 strip + 🔧 FAB 菜单 |
| `/play/:episodeId` | VideoPlayer  | 纯播放：顶栏 + 视频区 + OSC（含选集📋）+ 全屏 auto-hide |
| 覆盖层                | SearchOverlay | `/` 或 Ctrl+K 触发，居中搜索浮层，portal 注入 body             |
| 弹窗                 | Settings     | 侧边栏 6 标签页（媒体库 / 元数据 / 播放 / 通用 / 快捷键 / 关于），portal 注入 body |
| 覆盖层                | Onboarding   | 首次启动引导：创建推荐文件夹结构 / 直接进入设置                |

**全局背景**：`GlobalBackground`（App 级三层 fixed 浮层）通过 `useBackground().setBg()` 控制。浏览页完整氛围背景，播放页 gradient 透明 + fanart null，视频透底。

**全屏**：Tauri 原生 `set_fullscreen` + 统一布局条件样式。OSC/顶栏/光标 3s 无操作自动淡出。

**OSC 主题**：5 种可切换预设（Mochi/YouTube/PotPlayer/Netflix/极简），`src/themes/oscThemes.ts`，设置页选择器。

**选集面板**：始终 5 列网格布局（56×38 pills），当前集暖金 accent，已看绿色标识。超过 240px 高度纵向滚动。

**设置页**：左侧 140px 侧边栏 + AnimatePresence 内容切换。4 个子组件：媒体库（含元数据设置）/ 播放 / 通用 / 关于。批量拉取元数据后台运行，关闭设置页不中断。

**窗口关闭**：两套路径分别处理：

- 标题栏 ✕ → `invoke("window_close")` → `commands::window_close` 读 `close_behavior`
- Alt+F4 → `CloseRequested` 事件 → `lib.rs` 读 `close_behavior`
- "exit" 行为用 `std::process::exit(0)` 杀进程（Tauri 的 `app.exit(0)` 在 CloseRequested 内不可靠）
- 配置持久化于 `config.json`（路径由 `paths::data_root()` 决定，见下）

**数据路径 & 分发**：`.portable` 标记文件控制数据位置：

- exe 同级存在 `.portable` → 便携模式，所有数据（DB / config / cache）存 exe 旁边
- 不存在 → 安装模式，数据存 `%APPDATA%/mochi/`
- 核心模块 `src-tauri/src/paths.rs` 统一提供 `data_root()` 和 `cache_root()`
- 便携版 zip 打包脚本：`scripts/package-portable.ps1`（自动读版本号 + 创建 `.portable` 标记）
- NSIS 安装器：`pnpm tauri build --bundles nsis` → `target/release/bundle/nsis/Mochi_{version}_x64-setup.exe`

## 关键约束

**不可动：**

- mpv 单例，不销毁重建
- `refreshTracks` 不可用：`track-list` node 序列化在 libmpv-wrapper v0.1.1 + libmpv-2 v0.41.0 下崩溃
- resume seek 必须在 observer 收到 `duration` 回调后执行（文件完全加载前 seek 被忽略）
- `window_close` 命令和 `CloseRequested` 事件两套关闭路径必须同步维护
- `std::process::exit(0)` 用于"彻底退出"路径，不可改为 `app.exit(0)`（后者在 CloseRequested 内失效）

**元数据策略（v0.2.1 更新）：**

- **类型路由**：`anime` → Bangumi 优先；`tv` | `movie` | `variety` → TMDB 优先；`unknown` → 并行搜索。综艺不在 Bangumi 覆盖范围内，必须走 TMDB
- `build_bangumi_result` 不设 `tmdb_id` 和 `fanart_path`，由 `commands.rs` 外层统一补 TMDB 横幅
- Anime 无视已存储的 `tmdb_id`（防止 TMDB 兜底污染）
- `COALESCE` 保护：元数据返回 None 时 DB 保留已有数据。注意：poster/fanart 是 `COALESCE(?8, poster_path)`（新值优先），与其他字段语义一致（v0.2.1 修复：此前写反为 `COALESCE(poster_path, ?8)` 导致新海报无法覆盖旧缓存）
- Bangumi search `responseGroup=medium`：搜索结果自带摘要/评分/标签
- **演员表**：anime → Bangumi characters（角色图），tv/movie → TMDB credits（演员照片）
- **剧集信息**：统一走 TMDB season endpoint。anime 需额外 TMDB 搜索获取 tmdb_id（异步，不阻塞扫描）
- **剧集缩略图**：`w300`，本地缓存，路由切换时立即发起下载（与入场动画并行）
- **文件夹季号识别（v0.2.1）**：扫描器从文件夹名提取季号（`第X季` / `Season X` / `SX`，支持中文数字「六」→6），应用于该文件夹内所有文件的默认 season_number。SXXEYY 等显式文件名季号不被覆盖
- **季节级系列元数据（v0.2.1）**：`fetch_episode_metadata` 检测到所有剧集属于同一季时，从 TMDB season 接口拉取该季的海报、简介和名称，回写系列级 title（`节目名 + 季名`）、poster、synopsis

**v0.1.1 中修过的关键 Bug（防止回退）：**

- 离开播放页 → `handleBack` 必须 await 保存进度 + `cleanup` 加 `command("stop")`
- 自动连播不能走 `saveAndGo`（>95% progress 会误标 `watched_completed`）
- `WebView2Loader.dll` 必须与 exe 同级（Windows 加载 DLL 不搜索子目录）。make sure it's in `src-tauri/WebView2Loader.dll` (listed in `bundle.resources` at root level, NOT inside `lib/`)

**v0.2.1 新增约束：**

- 前端展示标题统一用 `series.title`（元数据拉取后的完整标题），不再用 `display_name`（扫描时的文件夹名）。涉及 PosterWall / SeriesDetail / MetadataVerdict 共 6 处
- `clear_all_metadata_ids` 不得将 `search_term` 设为 NULL（列有 NOT NULL 约束），v0.2.1 改为回退到 `display_name`
- `extract_season_from_name` 正则同时匹配 ASCII 数字和中文数字（`第([\d一二三四五六七八九十百千]+)季`），附带 `parse_chinese_numeral` 转换器
- `fetch_episode_metadata` 的 season poster 下载使用 `force: true`，确保覆盖 series 级海报缓存

**v0.2.4 新增约束：**

- NFO 解析是完全被动的旁路模块：不嵌入 `scanner.rs`，在 `commands.rs` 的 `scan_library` 和 `rescan_series_folder` 中以独立后处理步骤运行
- `db::apply_nfo_series_metadata` 使用 `COALESCE(existing, new)` 仅填充 NULL 字段，不覆盖已有在线元数据
- NFO 文件不存在或解析失败均静默跳过，不阻塞扫描流程
- 依赖 `quick-xml = "0.37"`，仅用于 `nfo.rs`

**v0.3.0 新增约束：**

- `SearchOverlay` 通过 `createPortal` 注入 `document.body`，脱离组件树层级

- `window_minimize` 命令已注册到 `generate_handler`，TitleBar 最小化按钮可用

- Indicator Dots 超过 7 部时窗口化显示，使用 `mask-image` 渐变淡出容器边缘

**v0.2.3 新增约束：**

- `refresh_single_series` 替代详情页原有的 `fetch_metadata` → `fetch_cast` → `fetch_episode_metadata` 三次调用链，统一为一个命令
- `fetch_cast` anime 分支三级兜底：有 bangumi_id → Bangumi，失败/无 ID → TMDB
- `try_tmdb` 新增 `prefer_animation` 参数（修改签名后需更新所有调用点：anime fallback 传 true，tv/movie/variety 和 unknown 传 false）
- `fetch_episode_metadata` 新增 `force` 参数（所有调用点需显式传入）
- - `clean_anime_folder_name` 在 `parse_folder_name` 之前调用，三个扫描路径均已接入
- ep_patterns 模式 1 已从 `_E(\d{1,3})` 扩展为 `(?:^|_)E(\d{1,3})([上下ab])?`，上下半集换算逻辑仅在 idx==0 触发
- `has_video_at_depth1` + 嵌套拆分逻辑：无视频 + 有子目录即拆为独立系列
- - `fullscreen` hook 初始化时调用 `get_fullscreen` 读真实状态
- NFO 元数据优先级低于在线数据：`rescan_series_folder` 和 `scan_library` 均在 upsert 后调用 `apply_nfo_series_metadata`（COALESCE 只填 NULL）

## 踩坑记录

1. **裸 `cargo build --release` 不走 Tauri 构建管线**：前端资源不嵌入，webview 尝试连接 devUrl。必须用 `npx tauri build`。

2. **自定义标题栏 ✕ 绕过 CloseRequested**：`TitleBar.tsx` 调用 `invoke("window_close")`，该命令原本写死 `window.hide()`，彻底退出永远不触发。修复：`window_close` 也读 `close_behavior`。

3. **Tauri app.exit(0) 在 CloseRequested 内不可靠**：事件循环状态导致退出信号丢失。改用 `std::process::exit(0)` 系统调用直接杀进程。

4. **ID 快速通道跳过 TMDB 横幅补拉**：已有 `bangumi_id` 的系列走 `fetch_by_bangumi_id`，该函数设 `fanart_path: None` 且不调 TMDB。修复：在 `commands.rs` 外层对所有 anime 结果统一补 TMDB backdrop。

5. **NSIS 安装器不自动包含 WebView2Loader.dll（gnu 工具链）**：Tauri NSIS 打包器未从 `target/release` 自动拾取该 DLL。修复：将 DLL 放入 `src-tauri/` 根目录，在 `bundle.resources` 中单独列出（不在 `lib/` 子目录，否则 Windows 加载不到）。
