## 当前架构

| 路由 | 组件 | 职责 |
|---|---|---|
| `/` | PosterWall | 海报横滚（响应式 vw/clamp）+ 筛选栏 + 标题 + 焦点圆点 |
| `/series/:id` | SeriesDetail | 系列信息 + 剧集列表 + ⋮ 菜单 + 元数据刷新动画 |
| `/play/:episodeId` | VideoPlayer | 纯播放：顶栏 + 视频区 + OSC（含选集📋）+ 全屏 auto-hide |
| 弹窗 | Settings | 侧边栏布局（5 section），portal 注入 body |

**全局背景**：`GlobalBackground`（App 级三层 fixed 浮层）通过 `useBackground().setBg()` 控制。浏览页完整氛围背景，播放页 gradient 透明 + fanart null，视频透底。

**全屏**：Tauri 原生 `set_fullscreen` + 统一布局条件样式。OSC/顶栏/光标 3s 无操作自动淡出。

**OSC 主题**：5 种可切换预设（Mochi/YouTube/PotPlayer/Netflix/极简），`src/themes/oscThemes.ts`，设置页选择器。

**选集面板**：始终 5 列网格布局（56×38 pills），当前集暖金 accent，已看绿色标识。超过 240px 高度纵向滚动。

**设置页**：左侧 140px 侧边栏 + AnimatePresence 内容切换。5 个子组件独立：媒体库 / 元数据 / 播放 / 通用 / 关于。

**窗口关闭**：两套路径分别处理：
- 标题栏 ✕ → `invoke("window_close")` → `commands::window_close` 读 `close_behavior`
- Alt+F4 → `CloseRequested` 事件 → `lib.rs` 读 `close_behavior`
- "exit" 行为用 `std::process::exit(0)` 杀进程（Tauri 的 `app.exit(0)` 在 CloseRequested 内不可靠）
- 配置持久化于 `config.json`（路径由 `paths::data_root()` 决定，见下）

**数据路径 & 分发**：`.portable` 标记文件控制数据位置：
- exe 同级存在 `.portable` → 便携模式，所有数据（DB / config / cache）存 exe 旁边
- 不存在 → 安装模式，数据存 `%APPDATA%/mochi/`
- 核心模块 `src-tauri/src/paths.rs` 统一提供 `data_root()` 和 `cache_root()`
- 便携版 zip 打包脚本：`scripts/package-portable.ps1`（自动读版本号 + 创建 `.portable` 标记）
- NSIS 安装器：`pnpm tauri build --bundles nsis` → `target/release/bundle/nsis/Mochi_{version}_x64-setup.exe`

## 关键约束

**不可动：**
- mpv 单例，不销毁重建
- `refreshTracks` 不可用：`track-list` node 序列化在 libmpv-wrapper v0.1.1 + libmpv-2 v0.41.0 下崩溃
- resume seek 必须在 observer 收到 `duration` 回调后执行（文件完全加载前 seek 被忽略）
- `window_close` 命令和 `CloseRequested` 事件两套关闭路径必须同步维护
- `std::process::exit(0)` 用于"彻底退出"路径，不可改为 `app.exit(0)`（后者在 CloseRequested 内失效）

**元数据策略：**
- `build_bangumi_result` 不设 `tmdb_id` 和 `fanart_path`，由 `commands.rs` 外层统一补 TMDB 横幅
- Anime 无视已存储的 `tmdb_id`（防止 TMDB 兜底污染）
- `COALESCE` 保护：元数据返回 None 时 DB 保留已有数据
- Bangumi search `responseGroup=medium`：搜索结果自带摘要/评分/标签

**v0.1.1 中修过的关键 Bug（防止回退）：**
- 离开播放页 → `handleBack` 必须 await 保存进度 + `cleanup` 加 `command("stop")`
- 自动连播不能走 `saveAndGo`（>95% progress 会误标 `watched_completed`）
- `WebView2Loader.dll` 必须与 exe 同级（Windows 加载 DLL 不搜索子目录）。make sure it's in `src-tauri/WebView2Loader.dll` (listed in `bundle.resources` at root level, NOT inside `lib/`)

## 本次对话踩坑记录

1. **裸 `cargo build --release` 不走 Tauri 构建管线**：前端资源不嵌入，webview 尝试连接 devUrl。必须用 `npx tauri build`。

2. **自定义标题栏 ✕ 绕过 CloseRequested**：`TitleBar.tsx` 调用 `invoke("window_close")`，该命令原本写死 `window.hide()`，彻底退出永远不触发。修复：`window_close` 也读 `close_behavior`。

3. **Tauri app.exit(0) 在 CloseRequested 内不可靠**：事件循环状态导致退出信号丢失。改用 `std::process::exit(0)` 系统调用直接杀进程。

4. **ID 快速通道跳过 TMDB 横幅补拉**：已有 `bangumi_id` 的系列走 `fetch_by_bangumi_id`，该函数设 `fanart_path: None` 且不调 TMDB。修复：在 `commands.rs` 外层对所有 anime 结果统一补 TMDB backdrop。

5. **NSIS 安装器不自动包含 WebView2Loader.dll（gnu 工具链）**：Tauri NSIS 打包器未从 `target/release` 自动拾取该 DLL。修复：将 DLL 放入 `src-tauri/` 根目录，在 `bundle.resources` 中单独列出（不在 `lib/` 子目录，否则 Windows 加载不到）。
