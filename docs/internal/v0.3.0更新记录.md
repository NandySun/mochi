# mochi v0.3.0 更新记录

## 概述

本版本聚焦首页浏览效率：**搜索**快速定位系列，**indicator dots 可交互化**提供精准跳转，**快捷键集中管理**统一分散在各处的快捷键参考。

---

## 一、搜索

### 1.1 搜索浮层

`SearchOverlay` 组件，`/` 或 `Ctrl+K` 触发，Esc 或点击遮罩关闭。居中浮层 420px 宽，`spring.gentle` 入场动画。底部一行键盘操作提示。

搜索范围：`title` + `search_term` + `year`，60ms debounce，最多显示 8 条结果。每条结果包含 36×54 缩略图 + 标题 + 类型标签 + 年份。↑↓ 键盘导航，Enter 进入详情页，hover 切换选中态。空结果时显示"未找到匹配的系列"占位。

`createPortal` 注入 `document.body`，底层 `backdrop-filter: blur(6px)` 保留首页氛围。搜索打开时首页轮播的 ←→ 键临时禁用，避免冲突。

**涉及文件**：`SearchOverlay.tsx`（新增）、`PosterWall.tsx`

### 1.2 搜索按钮

首页筛选栏右侧新增 🔍 搜索按钮（32×32 SVG 放大镜图标），与设置齿轮并排。点击行为与快捷键完全一致。确保不依赖快捷键的用户也能发现搜索功能。

**涉及文件**：`PosterWall.tsx`

---

## 二、首页 Indicator Dots 交互化

### 2.1 可点击跳转

每个圆点包裹 24×20px 透明 `<button>`，扩大点击热区。点击直接跳转到对应系列，`aria-label` 提供无障碍标识。

### 2.2 窗口化显示

超过 7 部时不再显示全部圆点，改为只渲染当前选中位 ±3 的 7 个。容器两端 `mask-image` 渐变淡出，暗示左右还有更多。7 部及以内全部显示，不做截断。

**涉及文件**：`PosterWall.tsx`

## 三、快捷键集中管理

### 3.1 SettingsShortcuts 页面

新建 `SettingsShortcuts` 组件，按页面分组列出全部快捷键：

| 分组 | 内容 |
|------|------|
| 全局 | `Ctrl+,` 打开设置、`/` `Ctrl+K` 搜索 |
| 首页 | ←→ 切换系列、Enter 进入详情、鼠标滚轮切换 |
| 搜索 | Esc 关闭、↑↓ 导航结果、Enter 打开 |
| 详情页 | Esc 关闭弹窗/返回 |
| 播放页 | Space 暂停、←→ 进退 5s、↑↓ 音量、F 全屏、Esc 退出、M 静音、[] 倍速、P/N 切集、J 字幕轨、双击全屏 |
| 弹窗 | Esc / 点击遮罩关闭 |

每组以 `label` 标题 + 浅色背景卡片承载，`<kbd>` 标签样式复用 `styles/settings.ts` 导出的 `kbdStyle`。

设置侧边栏新增第 6 个标签「快捷键」，插入在「通用」与「数据」之间。

**涉及文件**：`SettingsShortcuts.tsx`（新增）、`Settings.tsx`、`styles/settings.ts`

### 3.2 播放页快捷键参考移除

`SettingsPlayback` 中原有的播放器快捷键参考表移除。`kbdStyle` 定义从组件本地提取到 `styles/settings.ts` 供 `SettingsShortcuts` 和 `SettingsPlayback` 共用。`SettingsPlayback` 现在只负责 OSC 主题选择。

**涉及文件**：`SettingsPlayback.tsx`、`styles/settings.ts`

## 四、Bug 修复（2026-06-29）

### 4.1 标题栏最小化按钮失效

`commands.rs` 中 `window_minimize` 函数已定义，但 `lib.rs` 的 `generate_handler` 遗漏了注册。导致 TitleBar 的最小化按钮（─）无响应。补上注册后恢复正常。

**涉及文件**：`lib.rs`

### 4.2 详情页 Esc 返回首页

`App.tsx` 全局快捷键中补充 Escape 处理：当前路径非 `"/"` 时按 Esc 回到首页。覆盖详情页、播放页等子页面返回场景。

**涉及文件**：`App.tsx`
