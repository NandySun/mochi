# 音轨/字幕切换修复

> 待办 · 2026-06-29

---

## 问题

播放页点击切换字幕轨或音轨后，UI 不更新（轨道列表显示旧状态），且切换行为不可靠。

## 根因

`libmpv-wrapper v0.1.1` 中 `mpv_node` 结构体的序列化逻辑在读取 `track-list` 属性时崩溃。`refreshTracks()` 调用链：

```
getProperty("track-list/count") → libmpv-wrapper 序列化 track-list 节点 → 崩溃
```

crash 发生在 Rust FFI 层，前端 `catch { }` 吞掉了错误，UI 保持旧轨道列表。

## 影响范围

| 功能 | 状态 |
|------|------|
| OSC 字幕轨切换 | 切换指令发出但 UI 不更新 |
| OSC 音轨切换 | 同上 |
| OSC 轨道菜单展示 | 初次打开播放页时可能正常（外挂字幕由 `loadSubtitleFiles` 注入），但后续切换后菜单不更新 |
| 选集面板轨道标记 | 间接影响 |

## 可修复路径

1. **绕过 `refreshTracks` 的 track-list 读取**（推荐先行）
   - 前端维护轨道列表状态，不给 libmpv-wrapper 读 `track-list/count`
   - 通过 `observe_property("sid")` + `observe_property("aid")` 监听当前选中轨
   - 外挂字幕轨由 `loadSubtitleFiles` 的路径列表推算

2. **Fork libmpv-wrapper 修复序列化**
   - 库体量极小（17 commits，核心是 FFI 序列化）
   - 上游已休眠 7 个月，无同步负担
   - 修复后需替换 DLL 并验证

## 当前状态

- libmpv-wrapper: v0.1.1（最新，2025-11-23）
- tauri-plugin-libmpv: 0.3.2（最新）
- 上游 issue: 无公开报告
- 优先级：中（不影响核心播放，但影响多轨内容的体验）
