## 阶段一：Bug 修复（先修再改）

### 1.1 窗口拖动权限修复

**问题**：自定义标题栏 `data-tauri-drag-region` 已就位，但窗口无法拖动。

**根因**：`src-tauri/capabilities/default.json` 缺少 `core:window:allow-start-dragging` 权限。Tauri v2 中拖拽是受控能力，必须显式授权。

**步骤**：

1. 打开 `src-tauri/capabilities/default.json`
2. 在 `"permissions"` 数组中添加 `"core:window:allow-start-dragging"`
3. 无需改动 TitleBar.tsx（`data-tauri-drag-region` 已存在于 `<header>`）

```json
{
  "identifier": "default",
  "description": "Default capability for Mochi",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "core:window:allow-set-fullscreen",
    "core:window:allow-start-dragging",
    "libmpv:default",
    "shell:allow-open"
  ]
}
```

4. Build 验证：拖动标题栏左侧区域，"Mochi"文字和按钮之间空白区可拖拽移动窗口。

**备选**：如果加权限后仍无效（极低概率，与 WebView2 版本相关），改用 JS 方案——在 TitleBar 中 `import { getCurrentWindow } from "@tauri-apps/api/window"`，header `onMouseDown` 调用 `getCurrentWindow().startDragging()`。但权限方案是官方推荐路径，应优先采用。

## 阶段二：海报墙响应式缩放

**问题**：海报卡片（180×270px）、标题（24px）均为固定 px 值。窗口从默认 1200×800 放大到 1920×1080 或全屏时，海报和标题不跟随缩放，在画面中显得过小。

**方案**：用 `vw` + `clamp()` 纯 CSS 实现响应式，无需 JS 计算。

**改动文件**：`src/components/PosterWall.tsx`

### 2.1 PosterCard 尺寸响应式

```tsx
// 之前
width: 180,
height: 270,

// 之后
width: "clamp(110px, 15vw, 220px)",
height: "clamp(165px, 22.5vw, 330px)",
```

- 基准：1200px 宽窗口 → 15vw = 180px，行为同之前
- 下限：733px 宽窗口 → 110px（配合 minWidth:800）
- 上限：1467px 宽窗口以上 → 固定 220px（防止全屏时夸张放大）
- 宽高比恒为 2:3

### 2.2 标题字号响应式

```tsx
// 之前
fontSize: 24,

// 之后
fontSize: "clamp(14px, 2vw, 28px)",
```

### 2.3 标题纵向位置调整

当前标题位置基于固定 px 的链条：`top: calc(28% + 270px + 72px)`。海报高度改为 vw 后，这个链条失效。

改法：标题放在海报滚动容器内的最后（或在海报容器之后用 flex/margin），不依赖绝对定位的复杂 calc。

更简单的方案：在海报横滚容器下方渲染标题和焦点圆点，用 `margin-top` 控制间距：

```tsx
{/* Poster carousel */}
<div style={{ position: "absolute", top: "20%", left: 0, right: 0, zIndex: 1 }}>
  <div ref={containerRef} data-scroll style={{ /* 海报横滚 */ }}>
    {filtered.map(...)}
  </div>
  {/* 标题直接跟在海报容器内部末尾 */}
  {selected && (
    <div style={{
      textAlign: "center",
      marginTop: "clamp(8px, 1.5vw, 24px)",
      fontSize: "clamp(14px, 2vw, 28px)",
      fontWeight: 500,
      color: "rgba(255,255,255,0.7)",
      userSelect: "none",
    }}>
      {selected.display_name}
    </div>
  )}
  {/* 焦点圆点同理 */}
  {filtered.length > 1 && (
    <div style={{
      display: "flex", justifyContent: "center", gap: "clamp(4px, 0.5vw, 6px)",
      marginTop: "clamp(6px, 0.8vw, 12px)",
    }}>
      {filtered.map((_, i) => (...))}
    </div>
  )}
</div>
```

这样标题和圆点始终跟随海报容器，不需要计算复杂的绝对定位链条。

### 2.4 海报横向间距响应式

```tsx
// 之前
gap: 24,

// 之后
gap: "clamp(12px, 2vw, 32px)",
```

### 2.5 海报容器 padding 响应式

```tsx
// 之前
padding: "24px calc(50% - 90px)",

// 之后
padding: "clamp(12px, 2vw, 24px) calc(50% - clamp(55px, 7.5vw, 110px))",
```

让居中逻辑跟随海报半宽自适应。

### 2.6 首字母占位字号响应式

```tsx
// 之前（在 PosterCard 中）
fontSize: 64,

// 之后
fontSize: "clamp(36px, 5.3vw, 72px)",
```

## 阶段四：自动连播

**问题**：当前剧集播放结束后视频停止，需手动点击下一集。

**方案**：监听播放进度，当 `timePos >= duration - 3`（最后 3 秒）时自动跳转下一集。

**改动文件**：`src/components/VideoPlayer.tsx`

### 4.1 检测播放结束

```tsx
const autoNextTriggeredRef = useRef(false);

useEffect(() => {
  if (!mpv.duration || mpv.duration <= 0 || !nextEp) return;
  if (mpv.timePos >= mpv.duration - 3 && !autoNextTriggeredRef.current && !mpv.paused) {
    autoNextTriggeredRef.current = true;
    saveAndGo(nextEp.id);
  }
  // 切换剧集时重置
  if (mpv.timePos < 2) autoNextTriggeredRef.current = false;
}, [mpv.timePos, mpv.duration, nextEp, mpv.paused]);
```

### 4.2（可选）倒计时提示

在 OSC 上显示"下一集 · E04"的文字提示。可以复用现有的 now playing 位置或新增一个小标签。此为锦上添花，可跳过。

## 阶段五：窗口位置/大小记忆

**问题**：应用重启后窗口恢复默认位置和大小，不记得上次的调整。

**方案**：使用 `tauri-plugin-window-state`。

### 步骤

1. `src-tauri/Cargo.toml` 添加：
```toml
tauri-plugin-window-state = "2"
```

2. `src-tauri/src/lib.rs` 在 setup 中添加：
```rust
use tauri_plugin_window_state;

// 在 .setup(|app| { ... }) 的闭包开头添加：
app.handle().plugin(tauri_plugin_window_state::Builder::default().build())?;
```

3. `src-tauri/capabilities/default.json` 添加权限：
```json
"window-state:default"
```

插件会自动记住窗口的 position、size、maximized 状态，重启后恢复。

### 6.5 Esc 浮层优先关闭

**改动文件**：`src/components/OscBar.tsx` + `src/components/VideoPlayer.tsx`

当 OSC 的选集浮层、轨道菜单或音量滑块打开时，按 Esc 应先关闭浮层，而非直接返回详情页。

在 OscBar 中用 Context 或回调通知父组件是否有浮层打开：

最简单方案：OscBar 新增 prop `onEscapeCapture?: () => boolean`，VideoPlayer 的键盘 handler 中 Esc 分支改为：

```tsx
case "Escape":
  if (isFullscreen) {
    toggleFullscreen();
  } else if (oscEscapeRef.current?.()) {
    // OscBar 处理了（关闭了浮层），不继续冒泡
  } else {
    handleBack();
  }
  break;
```

OscBar 内部判断：若有任何浮层打开 → 关闭之 → 返回 true，否则返回 false。

更简单的方式：OscBar 在浮层打开时给 `document.body` 加一个 `data-osc-overlay` 属性，VideoPlayer 的 Esc handler 检查此属性决定行为。
