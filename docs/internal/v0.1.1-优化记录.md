## 执行中发现的 Bug 及修复

| Bug | 发现阶段 | 原因 | 修复 |
|---|---|---|---|
| 离开播放页后视频音频继续播放 | 用户测试 | `useMpv.cleanup()` 只 unlisten observer，不停止 mpv 播放 | cleanup 加 `command("stop", [])` |
| 自动连播标记剧集为"已完成" | 用户测试 | auto-next 走 `saveAndGo` 保存 ~97% progress，SQL `>95%` 触发 `watched_completed=1` | auto-next 改为直接 `goTo`，不保存进度 |
| 离开播放页后当前进度不保存 | 用户测试 | `handleBack` 未保存进度；Escape 快捷键闭包捕获过期 `mpv.timePos` | `handleBack` 改为 async + await 保存 + 使用 `timePosRef.current` |
| 详情页永不显示「继续」按钮 | 用户测试 | 前端 `episodes.find()` 不可靠 + 条件 `resumeEp.id !== firstEp?.id` 在第一集时退化 | 改用后端 `get_series_resume_episode` 命令 + 条件改为 `resumeEp` 存在即显示「继续」 |
| 点击「继续」后视频从 0:00 开始 | 用户测试 | resume seek 在 `loadFile` 之前执行，文件未加载时 seek 被忽略 | 改在 observer 收到 `duration` 事件后执行 seek（文件已完全加载） |

### Bug 1-3 详情：离开播放页进度丢失

**根因链**：
```
handleBack 未保存 → navigate → 组件卸载
  → cleanup 只 unlisten，不停止 mpv（单例约束）
  → 视频继续播放，但 observer 已注销，无人保存进度
  → DB 无最新进度 → 详情页找不到 resume
```

**修复**：
1. `handleBack` 改为 async，await 保存进度后再 navigate
2. handleBack 用 `timePosRef.current`（ref 实时值）替代 `mpv.timePos`（可能过期的闭包值）
3. `cleanup` 加 `command("stop", [])` 停止播放
4. auto-next 不再走 `saveAndGo`（避免 >95% 触发 completed 标记）

### Bug 4-5 详情：详情页续播不显示 / 不恢复进度

**根因**：前端 `episodes.find()` 匹配不到 resume 剧集（原因不明）。改用后端查询 `get_series_resume_episode` 后数据正常，但 resume seek 时机错了。

**修复**：
1. 新增 `get_series_resume_episode(series_id)` 后端命令
2. 详情页改用后端查询替代前端 find
3. resume seek 从 `loadFile` 之后移到 observer 的 `duration` 回调中（确保文件完全加载后再 seek）
4. 条件放宽：`resumeEp` 存在即显示「继续」，不管是否为第一集
