# v0.1.5 更新记录

> 2026-06-23 · 字幕切换 · OSC 图标 SVG 化 · 轨道枚举修复

## 字幕切换

### 外挂字幕路径存储（Rust 侧）

扫描器原本只记录 `subtitle_count` 数字，不存文件路径。现在 `EpisodeScan` 新增 `subtitle_paths: Vec<String>`，`count_subtitles` 在匹配外挂字幕时同步收集规范化路径。DB 新增 `subtitle_paths TEXT` 列（JSON 数组），幂等迁移，ON CONFLICT UPDATE 不覆盖播放进度。

### 轨道枚举修复

`refreshTracks` 原先调用 `getProperty("track-list", "node")` 在 libmpv-wrapper v0.1.1 + libmpv-2 v0.41.0 下崩溃。改为逐个属性枚举：

```
track-list/count → 循环 track-list/N/type / id / title / selected
```

标量属性不触发 node 序列化崩溃，轨道列表可正常获取。

### 轨道切换修复

`setSubTrack` / `setAudioTrack` 原先用 `setProperty("sid"/"aid", trackId)`，但 libmpv-wrapper 将所有 JS number 统一发成 `MPV_FORMAT_DOUBLE`，mpv 的 `sid` / `aid` 属性类型为 int64，收到 double 后静默拒绝。

改为 `command("set", ["sid"/"aid", String(trackId)])`，由 mpv 命令解析器自行处理类型转换。

### 外挂字幕加载

播放器 `loadFile` 后自动调用 `loadSubtitleFiles`（逐个 `sub-add`），800ms 和 1600ms 两个时间点分别刷新轨道列表，确保内封和外挂字幕都在 ⋮ 菜单中可见、可切换。

## OSC 图标 SVG 化

所有 emoji 按钮替换为自定义 SVG 图标组件（`src/components/icons.tsx`）：

| 按钮 | 旧 (emoji) | 新 (SVG) |
|---|---|---|
| 上一集 | ⏮ | `IconPrev` — 双左向折线 |
| 播放/暂停 | ▶ / ⏸ | `IconPlay` / `IconPause` — 三角形/双竖线（fill 实体） |
| 下一集 | ⏭ | `IconNext` — 双右向折线 |
| 音量 | 🔊 / 🔇 | `IconVolume` / `IconVolumeMute` — 喇叭 + 声波弧线/静音叉 |
| 选集 | 📋 | `IconEpisodes` — 2×2 圆角矩形网格 |

全部图标使用 `currentColor` 继承 OSC 文字色，2px stroke（播放/暂停用 fill），24×24 viewBox。不受系统 emoji 字体差异影响。

### CC 盲切按钮移除

CC 按钮（盲切字幕）从 OSC 移除。`j` 键保留为字幕盲切快捷键。

## Bug 修复

### 音量 knob 偏移

音量滑块的白点 knob 在帧运动（`animate={{ scale }}`）下，framer-motion 的 `style.transform` 覆盖了 inline style 的 `translate(-50%, -50%)`，导致 `translateX(-50%)` 丢失，knob 水平偏右 4px。

修复：将 `translate` 从 inline style 移入 framer-motion 的 `animate` 对象（`x` / `y` 属性），由 framer-motion 内部合成完整的 transform 字符串。进度条 knob 同修。

## 2026-06-23 第二轮 · 设置页重构 · 批量拉取后台化 · 分类扩展 · 交互打磨

### 设置页分类重构

`SettingsMetadata.tsx` 内容合并进 `SettingsMedia.tsx`，原文件不再被引用。

### 批量拉取元数据后台化

问题：设置页的「批量拉取全部元数据」在关闭设置页后进程中断（组件卸载导致 React 状态更新失效，用户看不到进度和结果）。

方案：async 函数一旦被调用就不会因组件卸载而中断（JS 闭包保持 alive），关键改动是进度/完成通知从 React setState 改为 `window.dispatchEvent` 自定义事件：

- `mochi:batch-fetch-start`
- `mochi:batch-fetch-progress` (detail: `{ current, total }`)
- `mochi:batch-fetch-complete`

`App.tsx` 监听 `mochi:batch-fetch-complete`，完成后从顶部滑入绿色 banner「元数据拉取完成 ✓」，4 秒自动消失。

防重：`localStorage` 标记位 `mochi_batch_fetch_running`，关闭重开设置页时自动恢复「后台拉取中…」状态，不允许二次触发。

### 首页分类过滤器扩展

新增「电影」和「综艺」两个过滤器标签。

Rust 扫描器同步扩展：`FOLDER_SUFFIX_RE` 正则增加 `variety`、两处 `TYPE_HINTS` 增加 `("variety", "variety")`，`D:\Video\variety\` 下的文件夹自动识别为综艺类型。

TypeScript 侧 `Series.type` / `MetadataResult.series_type` 联合类型、详情页类型下拉、`PosterWall` 过滤器 state 全部同步扩展。

修复了此前 `movie` 类型系列在首页「全部」以外标签不可见的问题（过滤器原来只有 anime / tv）。

### 详情页简介滚动限定

简介区域限定 `maxHeight: 120px`，超出可滚动。滚动条 WebKit 定制：4px 宽、轨道透明、滑块 `rgba(255,255,255,0.08)`，hover 时略亮。`scrollbarWidth: thin` + `scrollbarColor` 保留 Firefox 兼容。

### 演员表横滚

CastStrip 原先只有 `overflowX: auto`，鼠标滚轮无法触发横向滚动。增加 native wheel 事件监听（`{ passive: false }`，`deltaY * 2.5` 转横向 `scrollBy`），与首页海报墙、剧集 strip 统一滚动策略。

## 演员表

### 数据模型

新建 `person` 表和 `series_cast` 关联表：

- `person(source, source_id, name, role_name, image_url, image_cache)` — `(source, source_id)` 唯一约束
- `series_cast(series_id, person_id, sort_order)` — 复合主键

### 数据路由

| 类型 | 源 | 主名 | 副名 | 头像 |
|---|---|---|---|---|
| anime | Bangumi `/v0/subjects/{id}/characters` | 角色名 | 声优名 | 角色图 (grid) |
| tv/movie | TMDB `/{type}/{id}/credits` | 演员名 | 角色名 | 演员照片 (w185) |

Bangumi 角色头像下载到 `bangumi_char_{id}.jpg`，TMDB 演员头像下载到 `tmdb_{id}_cast.jpg`。

### CastStrip 组件

48px 圆形头像横滚列表，`hover scale(1.12)` + 暖金 glow，首字母占位圆。无数据时整个区域不渲染。

### 拉取策略

元数据刷新（详情页 ⋮ 或设置页批量拉取）自动串联演员表 + 剧集信息的拉取，一次 `刷新元数据` 完成全部三类数据获取。

## Rust 侧新增模块

### `metadata/episodes.rs`

5 阶段分离架构：DB 读（获取 series + episodes）→ 释放锁 → TMDB 网络请求（season detail + 缩略图下载）→ 重获锁 → DB 写。解决 Tauri async command 的 `Send` trait 约束。

### `metadata/cast.rs`

同构分离架构。`fetch_cast` 按 series.type 路由 Bangumi `get_characters()` 或 TMDB `get_credits()`，Person 批量 upsert + `replace_series_cast`。

### TMDB 扩展

- `TmdbClient::get_season(tv_id, season_number, language)` → `TmdbSeasonDetail`
- `TmdbClient::get_credits(media_type, id, language)` → `TmdbCreditsResponse`
- 新增类型：`TmdbEpisodeResult` / `TmdbCastMember`

### Bangumi 扩展

- `BangumiClient::get_characters(subject_id)` → `Vec<BangumiCharacter>`
- 新增类型：`BangumiCharacter` / `BangumiCharacterActor`

### 新命令

| 命令 | 参数 | 说明 |
|---|---|---|
| `fetch_episode_metadata` | seriesId, tmdb_api_key | 拉取剧集缩略图/名称 |
| `fetch_cast` | seriesId, tmdb_api_key | 拉取演员表 |
| `get_series_cast` | seriesId | 读取已存储演员表 |

全部接受 `tmdbApiKey` 参数（前端从 localStorage 传入），不再依赖 `env::var`。
