## P0 — 立即修，成本极低

### 5. 托盘左键单击打开窗口

**文件**：`src-tauri/src/lib.rs`

**改动**：在 `TrayIconBuilder` 链上追加 `on_tray_icon_event`：

```rust
.on_tray_icon_event(|tray, event| {
    if let tauri::tray::TrayIconEvent::Click {
        button: tauri::tray::MouseButton::Left,
        ..
    } = event {
        let app = tray.app_handle();
        if let Some(window) = app.get_webview_window("main") {
            let _ = window.show();
            let _ = window.set_focus();
        }
    }
})
```

**交互逻辑**：左键 → 显示+聚焦；右键 → 保持现有菜单。不做 toggle（不隐藏已显示的窗口）。

### 6. 版本号自动化

**问题**：`Settings.tsx` 硬编码 `Mochi v0.1.0`，`tauri.conf.json` 写 `0.1.2`，`package.json` 写 `0.1.2`。三处独立维护，必然漂移。

**方案**：以 `tauri.conf.json` 为单一真相源。

**Rust 侧**（`commands.rs` 新增）：

```rust
#[tauri::command]
fn get_app_version(app: tauri::AppHandle) -> String {
    app.package_info().version.to_string()
}
```

**前端侧**（`Settings.tsx` 关于 section）：用 `invoke("get_app_version")` 替换硬编码。

**关于页内容规划**（参考 HanaAgent 风格）：

- 应用名 + 版本号（动态获取）
- 一句话描述："桌面原生个人多媒体库"
- 版权：`© 2025-2026 Revm`
- 技术栈：Tauri v2 · React 19 · libmpv · SQLite · Bangumi · TMDB
- GitHub 仓库链接
- "检查更新"按钮（轻量版：打开 GitHub Releases 页面；不做 API 轮询）
- 致谢：mpv / Tauri / Bangumi / TMDB
- 许可证：MIT + libmpv LGPL 说明

**具体实现见下文 P1-2 的 ⋮ 菜单方案**，因为刷新按钮将移入 ⋮ 菜单，加载动画随之调整。

## P1 — 交互优化，影响日常使用频率

### 2. 详情页元数据按钮 → ⋮ 菜单（含完整设计决策）

**问题**：当前"刷新元数据"和"✎ 编辑搜索词"以 11px 低透明度文字裸露在 genre 标签下方，视觉权重极低，布局突兀。

**设计决策**（本次会话讨论确定）：

#### 位置

⋮ 按钮放在 **类型选择器右侧**，与类型 dropdown 形成"元数据管理"工具组。布局：

```
类型 [动漫 ▾]  ⋮
```

#### 菜单样式

```
┌──────────────────┐
│ 🔄 刷新元数据     │
│ ✏️ 编辑搜索词     │
└──────────────────┘
```

- **图标 + 文字**（非纯图标非纯文字）：图标提供视觉锚点，文字保证语义明确
- 菜单项样式参考 OSC 轨道菜单：hover 时 `background: rgba(255,255,255,0.06)`，选中态用 `•` 标记（如搜索词编辑模式激活时，该项显示选中态）

#### 弹出动画

沿用 OscBar 的弹出动画事实标准：

```tsx
initial={{ opacity: 0, y: 8, scale: 0.96 }}
animate={{ opacity: 1, y: 0, scale: 1 }}
exit={{ opacity: 0, y: 8, scale: 0.96 }}
transition={spring.gentle}
```

菜单在 ⋮ 按钮下方 `calc(100% + 4px)` 处展开，右对齐（`right: 0`）。

#### 元数据刷新加载态

**⋮ 按钮本身变成 BreathingDot**，完整交互时序：

```
用户看到 ⋮ → 点击
  ↓
菜单弹出（spring.gentle, y: 8 → 0, scale: 0.96 → 1）
  ↓
点击"刷新元数据"
  ↓
菜单关闭（exit 动画）
  ↓
⋮ 变成 BreathingDot（size=16, 暖金色 #c47e3a）
  ↓（同步）
评分/简介区上方淡入 "正在拉取元数据…" 小字（fontSize: 11, opacity 淡入）
  ↓（3~15秒后）
数据更新 → BreathingDot 变回 ⋮
  ↓
加载提示消失，新数据渲染
```

如果编辑搜索词在刷新时是展开状态，刷新完成后自动收起。

#### 编辑搜索词的交互

- 点击菜单"编辑搜索词" → 菜单关闭 → 搜索词输入框在 ⋮ 按钮下方或旁边展开（inline），与现有逻辑一致
- 输入框自动 focus
- 输入框右侧有"确认"（触发刷新）和"取消"按钮，或者按 Enter 触发刷新、Esc 取消
- 编辑搜索词模式激活时，⋮ 按钮可以用一个视觉变化（如 accent 色边框）提示当前处于编辑状态

#### 代码实现要点

- `SeriesDetail.tsx` 中新增 `showKebabMenu` 状态
- 菜单用 `AnimatePresence` 包裹
- 菜单外点击关闭（`mousedown` 监听，与类型 dropdown 逻辑一致）
- `refreshingMeta` 状态下 ⋮ 渲染为 `<BreathingDot size={16} />` + tooltip "刷新中…"

### 4. 选集面板设计语言重塑

**问题**：OSC 的选集弹出面板是 360px 横向滚动条 + 40×34 编号按钮，没有融入 mochi 设计语言。

**mochi 设计语言特征**（从现有代码提炼）：

- 暖金 accent `#c47e3a`
- 大圆角（8-14px）
- 半透明深色背景 + `backdrop-filter: blur(12px)`
- 低透明度白色边框 `rgba(255,255,255,0.08)`
- BreathingDot 加载态
- framer-motion spring 动画（`spring.gentle`）
- 按钮 `whileHover` / `whileTap` 微交互
- 类型选择器选中态小圆点

**重塑方向**：

1. **面板容器**：应用毛玻璃面板样式，圆角 12-14px，`border: 1px solid rgba(255,255,255,0.08)`，与 OSC 轨道菜单视觉一致
2. **剧集按钮**：改用与详情页剧集列表一致的风格（56×38 pill 按钮），当前播放集边框/背景用 `#c47e3a`，已看集用 `rgba(90,170,150,0.12)`
3. **布局**：剧集数 > 50 时考虑网格布局（3-4 列），而非纯横向单行；≤ 50 保持横向滚动
4. **滚动体验**：两侧渐变遮罩提示可滚动区域 + 当前视口位置指示器
5. **动画**：保持现有 `spring.gentle` 弹出动画
6. **可选增强**：hover 时浮出剧集标题 tooltip（如果有）

**文件**：`src/components/OscBar.tsx` 中 `EpisodeBtn` 部分的 `showEpisodes` 弹出块。

### 13. 关闭窗口行为可选（托盘/退出）

**问题**：当前 ✕ / Alt+F4 无条件隐藏到托盘。用户希望可选择"彻底退出"。

**设计决策**（本次会话讨论确定）：

#### 选项

| 选项 | ✕ / Alt+F4 行为 | 默认 |
|---|---|---|
| 最小化到系统托盘 | 隐藏窗口，进程保留 | ✓ |
| 彻底退出程序 | 关闭窗口，进程退出 | |

#### 设置 UI

在设置页新增"通用" section（或放在现有 section 中），用与类型选择器一致的下拉/radio 选择：

```
关闭窗口行为
○ 最小化到系统托盘
○ 彻底退出程序
```

#### 实现架构

**配置文件**：`%APPDATA%/mochi/config.json`（与 `mochi.db` 同级）：

```json
{
  "close_behavior": "tray"
}
```

不用 localStorage 的原因：Rust 的 `CloseRequested` handler 在窗口事件循环中，无法直接读前端存储。JSON 文件 Rust 和前端都能读写。

**Rust 侧**：

1. `AppState` 加 `close_behavior: Mutex<String>`，启动时从 `config.json` 读（默认 `"tray"`）
2. `CloseRequested` handler 判断：
   - `"tray"` → `prevent_close()` + `hide()`
   - `"exit"` → 不做任何阻止，正常关闭
3. 新增两个 commands：
   - `get_close_behavior` → 读 config
   - `set_close_behavior` → 写 config + 更新 AppState

**前端侧**：设置页新增选择器，调用 `set_close_behavior` command。

**安全出口**：托盘右键菜单的"退出"始终真正退出，不受设置影响。

**进度保存**："彻底退出"时 `beforeunload` 事件仍会触发（VideoPlayer 已监听），播放进度不会丢失。

### 1. 设置页分页

**当前**：单文件 `Settings.tsx`（~340 行），四个 section 堆在一个滚动容器。

**方案**：侧边栏 + 内容区布局（HanaAgent 设置同款）：

```
┌────────┬──────────────────────────┐
```

- `Settings.tsx` 降级为布局容器 + 路由状态管理

**分页逻辑**（建议 section 划分）：

## P3 — 锦上添花

### 7. 设计系统落地一致性

**观察**：mochi 已有设计 token 雏形（`animations/tokens.ts`、`themes/oscThemes.ts`、`BreathingDot`），但落地不彻底。同一 BreathingDot，批量拉取用了，单个刷新没用；毛玻璃面板样式在 OSC 菜单用了，选集面板没用。

**建议**：加新 UI 时先问"有没有现成的组件/样式可以复用"。不需要现在就做完整 design system。

---

### 9. 键盘快捷键可发现性

**问题**：播放器有完整快捷键体系（Space / ←→ / ↑↓ / F / M / P / N / [ ]），但无任何入口可查看。

**建议**：
- 方案 A：播放器按 `?` 弹出快捷键参考面板（YouTube/B站 交互范式）
- 方案 B：在设置的"播放"或"通用" section 列出快捷键表
- 两个方案可以共存

---

### 10. 缓存管理可见

**问题**：元数据缓存在 `%APPDATA%/mochi/cache/` 持续增长，用户不可见也不可管理。

**建议**：在设置页加"缓存管理"项，显示缓存大小 + "清除缓存"按钮。需要 Rust command 计算目录大小。

---

### 11. 元数据刷新覆盖逻辑提示

**问题**：`handleRefreshMeta` 传 `force: false`（不强制覆盖已有字段），但 UI 未区分"补充"和"强制覆盖"。用户可能困惑为什么刷新了但简介没更新。

**建议**：刷新按钮 tooltip 或菜单项说明当前的覆盖策略。或者提供两个选项："补充缺失字段" / "强制覆盖全部"。

---

### 12. 首次关闭提示

**问题**：新用户点 ✕ 后窗口消失但进程还在，可能以为程序卡死了。

**建议**：首次关闭时做一个短暂 toast："Mochi 已最小化到系统托盘，右键图标可退出"。用 localStorage 记录是否已提示过，只显示一次。
