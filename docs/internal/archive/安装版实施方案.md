# Mochi — 安装版实施方案

> 2026-06-22 · 基于与 Hanako 讨论制定
> 状态：待执行

---

## 一、背景

mochi 当前仅有便携版（zip 发布），数据硬编码在 `%APPDATA%/mochi/`。本次改动同时实现：

- **安装版**（NSIS 安装器）：数据存 `%APPDATA%`，有开始菜单和卸载入口
- **便携版**（zip）：通过 `.portable` 标记文件让数据跟随 exe

两种分发形式从同一套代码产出，用户按需选择。

---

## 二、核心设计：`.portable` 标记 + 统一 `data_root()`

### 2.1 判定逻辑

```
启动时 → 检查 exe 同级目录是否存在 .portable 文件
├── 存在 → 便携模式：数据根目录 = exe 所在目录
└── 不存在 → 安装模式：数据根目录 = %APPDATA%/mochi/
```

### 2.2 数据布局对比

| 文件 | 便携模式 | 安装模式 |
|---|---|---|
| 数据库 | `<exe_dir>/mochi.db` | `%APPDATA%/mochi/mochi.db` |
| 配置 | `<exe_dir>/config.json` | `%APPDATA%/mochi/config.json` |
| 图片缓存 | `<exe_dir>/cache/` | `%APPDATA%/mochi/cache/` |

### 2.3 现状

当前三个函数各自硬编码 `%APPDATA%`：

| 函数 | 文件 | 当前行为 |
|---|---|---|
| `dirs_next()` | `lib.rs` L168-173 | 直接拼 `%APPDATA%/mochi/mochi.db` |
| `config_path()` | `lib.rs` L214-220 | 直接拼 `%APPDATA%/mochi/config.json` |
| `cache_dir()` | `metadata/cache.rs` L12-17 | 直接拼 `%APPDATA%/mochi/cache/` |

## 三、实施步骤

### Step 1：新建 `src-tauri/src/paths.rs`

抽取统一的数据根目录判定逻辑。所有路径函数引用此模块。

```rust
//! Platform-aware data directory resolution.
//!
//! Portable mode (`.portable` marker next to exe): data lives alongside the binary.
//! Installed mode (no marker): data lives in %APPDATA%/mochi/.

use std::path::PathBuf;

/// Return the data root directory, creating it if needed.
///
/// # Portable mode detection
/// If a `.portable` file exists in the same directory as the executable,
/// data is stored alongside the exe. Otherwise, `%APPDATA%/mochi/` is used.
pub fn data_root() -> Result<PathBuf, String> {
    let exe_dir = std::env::current_exe()
        .map_err(|e| format!("无法获取 exe 路径: {e}"))?
        .parent()
        .ok_or("exe 路径没有父目录")?
        .to_path_buf();

    let root = if exe_dir.join(".portable").exists() {
        exe_dir
    } else {
        let appdata = std::env::var("APPDATA")
            .map_err(|_| "APPDATA 环境变量未设置".to_string())?;
        PathBuf::from(&appdata).join("mochi")
    };

    std::fs::create_dir_all(&root)
        .map_err(|e| format!("无法创建数据目录: {e}"))?;

    Ok(root)
}

/// Convenience: data_root()/cache/
pub fn cache_root() -> Result<PathBuf, String> {
    let dir = data_root()?.join("cache");
    std::fs::create_dir_all(&dir)
        .map_err(|e| format!("无法创建缓存目录: {e}"))?;
    Ok(dir)
}
```

### Step 2：改 `lib.rs`

改动两个函数，都切换到 `paths::data_root()`：

```rust
// 文件头部加模块声明
mod paths;

// dirs_next() → 改为基于 data_root()
fn dirs_next() -> Option<String> {
    let db_path = paths::data_root().ok()?.join("mochi.db");
    Some(db_path.to_string_lossy().to_string())
}

// config_path() → 同样基于 data_root()
fn config_path() -> Result<PathBuf, String> {
    let path = paths::data_root()?.join("config.json");
    Ok(path)
}
```

**不影响其他代码。** `dirs_next()` 和 `config_path()` 是私有函数，只有 `run()` 内部调用，返回值格式不变。

### Step 3：改 `metadata/cache.rs`

`cache_dir()` 切换到 `paths::cache_root()`：

```rust
use crate::paths;

pub fn cache_dir() -> Result<PathBuf, String> {
    paths::cache_root()
}
```

其他函数（`anilist_cache_path`、`tmdb_cache_path`、`bangumi_cache_path`）都基于 `cache_dir()`，不需要改动。

### Step 4：改 `tauri.conf.json`

当前 `bundle` 段：

```json
"bundle": {
  "resources": ["lib/**/*"]
}
```

改为：

```json
"bundle": {
  "resources": ["lib/**/*"],
  "targets": ["nsis"],
  "createUpdaterArtifacts": false,
  "windows": {
    "nsis": {
      "installMode": "currentUser",
      "startMenuFolder": "Mochi",
      "displayLanguageSelector": false,
      "compression": "lzma"
    }
  }
}
```

| 配置项 | 值 | 说明 |
|---|---|---|
| `installMode` | `currentUser` | 不需要管理员权限，装到 `%LOCALAPPDATA%/Programs/Mochi/` |
| `startMenuFolder` | `Mochi` | 开始菜单文件夹名 |
| `displayLanguageSelector` | `false` | 不显示语言选择器（目前只做中文） |
| `compression` | `lzma` | 最高压缩比 |
| `createUpdaterArtifacts` | `false` | 暂不做自动更新 |

> **关于快捷方式**：Tauri v2 NSIS 默认就会在安装时询问是否创建桌面快捷方式和开始菜单快捷方式（内置在 NSIS 模板中），不需要额外配置。用户安装时可按需勾选。

### Step 5：便携版 zip 打包脚本

便携版 zip 需要包含 `.portable` 标记文件。使用 `scripts/package-portable.ps1`：

```powershell
# 先构建
pnpm tauri build

# 再打包便携版 zip
.\scripts\package-portable.ps1
# 产出：releases/mochi-v0.1.3-portable.zip（含 .portable 标记文件）
```

脚本自动从 `package.json` 读取版本号，收集 exe + DLL + `.portable` 标记，打包成 zip。

### Step 6：Asset Protocol Scope

当前 `assetProtocol.scope`：

```json
"scope": ["$APPDATA/mochi/cache/**", "$APPDATA/mochi/**"]
```

**不做修改。** 原因：前端图片加载通过 `read_image_base64` 命令直接读文件返回 data URL（见 `useImageSrc.ts`），不依赖 asset protocol。便携模式下 scope 失效不会影响功能。如果未来想启用 asset protocol 优化，再做动态 scope。

# 手动打包 zip 时记得带上 .portable 文件

## 五、测试验证

| 测试项 | 方法 | 期望结果 |
|---|---|---|
| 便携模式数据路径 | 解压 zip（含 `.portable`），启动 mochi | `mochi.db`、`config.json`、`cache/` 出现在 exe 同级目录 |
| 便携模式隔离性 | 换一个目录再解压一份，启动 | 第二份拥有独立的数据库和配置 |
| 安装版数据路径 | 运行 NSIS 安装器，安装后启动 | 数据出现在 `%APPDATA%/mochi/` |
| 安装版卸载 | 控制面板 → 卸载 Mochi | 程序文件删除（数据目录是否保留取决于是否在安装目录；NSIS `currentUser` 默认不删用户数据） |
| 窗口关闭行为 | 便携版和安装版分别设置关闭行为 | `config.json` 在各自的数据目录下独立生效 |
| 已有用户迁移 | 先启动安装版（数据在 `%APPDATA%`），再换便携版 | 便携版有独立的空白数据库，不影响安装版数据 |
