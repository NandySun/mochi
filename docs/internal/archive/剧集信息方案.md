# 剧集信息统一方案：TMDB Season

> 状态：**已确认**  
> 日期：2026-06-23  
> 确认日期：2026-06-23  
> 关联：演员表方案.md（平行讨论）

---

## 决策摘要

| 决策项 | 结论 |
|--------|------|
| 数据源 | 所有类型统一走 TMDB season endpoint |
| anime TMDB ID | Bangumi 匹配后额外搜 TMDB 获取（异步，不阻塞扫描） |
| 展示形式 | 详情页横滚 strip（180×101 卡片）+ 居中 modal（3 列缩略图网格） |
| 缩略图加载 | 本地缓存优先，路由切换时立即发起下载，与入场动画并行 |
| 缩略图尺寸 | `w300`（strip 180px / modal ~200px 渲染） |
| 缩略图缺失 | 暖金色纯色占位 + 集数编号（和海报占位策略一致） |
| Bangumi fallback | 不做。季/集编号体系不一致，混用反而混乱 |
| episode overview | 存入 DB，前端暂不展示 |

---

## 1. 背景

当前 mochi 的元数据覆盖止于 series 级别（标题、年份、简介、类型、评分、海报、横幅）。剧集（episode）级别目前只有扫描器从文件名正则提取的 episode_number，没有剧集名称，没有剧集缩略图。

经过对两个元数据源的 API 实测：

| 数据 | TMDB | Bangumi |
|------|------|---------|
| 剧集名称 | ✅ `/tv/{id}/season/{n}` → `name` | ✅ `/v0/episodes?subject_id=` → `name_cn` |
| 剧集缩略图 | ✅ `still_path` → 拼接 TMDB 图片 URL | ❌ episode schema 无图片字段 |

Bangumi 无法提供剧集缩略图，TMDB 两者都有。决定统一走 TMDB。

---

## 2. 现状

### 2.1 已有能力

- `TmdbClient` 已实现 search、detail（基本信息和类型标签）
- `MetadataResult` 有 `tmdb_id` 字段
- 对 tv/movie 类型，`tmdb_id` 在元数据拉取时写入（`build_tmdb_result` 填入 `Some(result.id)`）
- 图片缓存基础设施（`download_image` + `cache_dir`）已就绪

### 2.2 缺失

- **TMDB season endpoint 未接入**：`/tv/{id}/season/{season_number}` 未调用
- **anime 类型无 tmdb_id**：当前 anime 走 Bangumi 优先，`build_bangumi_result` 设置 `tmdb_id: None`。即使后续 `try_tmdb_backdrop` 成功拿了横幅图，也没有把 tmdb_id 写回 series 记录
- **剧集表无名称/缩略图字段**：SQLite `episodes` 表没有 `title` 和 `still_path` 列
- **剧集级别的元数据拉取流程**：完全不存在

## 3. TMDB Season API

### 3.1 端点

```
GET /tv/{series_id}/season/{season_number}
  ?api_key=xxx
  &language=zh-CN
```

### 3.2 返回结构（精简）

```json
{
  "episodes": [
    {
      "episode_number": 1,
      "name": "凛冬将至",
      "overview": "一名守夜人军团的逃兵...",
      "still_path": "/o4IX9Mm0kpLITVANJMx7inyEUaY.jpg",
      "air_date": "2011-04-17",
      "runtime": 62,
      "vote_average": 8.1
    }
  ]
}
```

### 3.3 缩略图拼接

```
https://image.tmdb.org/t/p/w300/{still_path}
```

尺寸用 `w300`（剧集缩略图展示尺寸，比 poster 的 `w500` 小）。

### 3.4 费用与限制

- 免费，和现有 search/detail 共用同一个 API key
- 无额外速率限制
- 不需要 TMDB 付费计划

## 4. 核心方案

### 4.1 总原则

**所有类型的 series，只要持有有效的 tmdb_id，统一从 TMDB season endpoint 拉取剧集名称和缩略图。**

不分 anime / tv / movie。数据源统一。

### 4.2 关键前置条件：anime 类型需要 tmdb_id

当前 anime 的 tmdb_id 为空。需要拓展元数据拉取流程，在 anime 路径下也填充 tmdb_id。

**策略：在 Bangumi 匹配成功后，额外做一次 TMDB 搜索，拿 tmdb_id。**

```text
anime 元数据拉取流程（修改后）：
  try_bangumi(search_term) → bangumi_result
    ↓ 成功
  try_tmdb_search(search_term) → tmdb_id（仅拿 ID，不拉完整元数据）
    ↓ 成功 → 写入 bangumi_result.tmdb_id
  try_tmdb_backdrop（已有逻辑，补横幅）
    ↓
  返回 MetadataResult（此时 tmdb_id 有值）
```

**为什么不在 anime 路径下做完整的 TMDB 拉取（build_tmdb_result）？**

因为 Bangumi 的中文元数据质量更高。标题、简介、标签都优先用 Bangumi 的。TMDB 只负责填补 Bangumi 没有的数据：横幅图 + 剧集名称 + 剧集缩略图。保持分层策略的一致性。

### 4.3 TMDB 覆盖度与回退

**不是所有 Bangumi 上的动画在 TMDB 都有对应条目。**

- 日本 TV 动画 → TMDB 覆盖度高（大多数有英文条目，部分有中文翻译）
- 剧场版/OVA → 覆盖度中等
- Web 动画、短片、冷门作品 → 可能找不到

**回退策略**：

```
拉取剧集信息：
  if tmdb_id is Some:
    call TMDB season endpoint
    if 成功 → 写 episode.title + episode.still_path
    if 失败 → episode.title 留空，无缩略图（静默降级）
  else:
    跳过，episode.title 留空，无缩略图
```

不做复杂的备选链（比如 TMDB 失败时回退 Bangumi）。保持简单：有 TMDB ID 就拉，没有就空着。Bangumi 的 episode name 也不做 fallback，因为两个数据源的 season/episode 编号体系可能不一致，混在一起反而出错。

## 5. 数据模型变更

### 5.1 episodes 表扩展

当前 episodes 表字段（来自 `scanner.rs` 的 `EpisodeScan`）：

```
episode_number, season_number, file_path, file_name,
subtitle_count, status, match_method
```

新增字段：

```sql
ALTER TABLE episodes ADD COLUMN title TEXT;          -- 剧集名称（来自 TMDB）
ALTER TABLE episodes ADD COLUMN still_path TEXT;     -- 剧集缩略图本地缓存路径
ALTER TABLE episodes ADD COLUMN still_url TEXT;      -- 剧集缩略图 TMDB 原始 URL（用于重新下载判断）
ALTER TABLE episodes ADD COLUMN overview TEXT;        -- 剧集简介（可选）
ALTER TABLE episodes ADD COLUMN air_date TEXT;        -- 上映日期（可选）
ALTER TABLE episodes ADD COLUMN runtime INTEGER;      -- 时长（秒）（可选）
ALTER TABLE episodes ADD COLUMN fetched_at TEXT;      -- 上次拉取时间戳
```

`still_path` 存本地缓存路径（下载后），`still_url` 存 TMDB 原始 URL，用于判断是否需要重新下载（URL 变了 = TMDB 换了图）。

### 5.2 不新增独立表

剧集元数据直接挂在 episodes 表上，不建独立的 episode_metadata 表。理由：
- 每个 episode 只有一份剧集元数据（来自 TMDB），不是多对多关系
- 减少 JOIN，查询简单
- 如果将来接入多个源（如 Bangumi episodes 作为 fallback），再拆分不迟

## 6. 拉取流程

### 6.1 触发时机

| 时机 | 行为 |
|------|------|
| series 元数据拉取完成 | 如果 `tmdb_id` 有值，异步触发剧集信息拉取 |
| 手动刷新元数据 | 连带刷新剧集信息 |
| 页面上浏览剧集列表 | 按需拉取（如果已缓存则直接展示） |

核心原则和演员表一致：**剧集信息是增值数据，不阻塞基础元数据和扫描结果的展示。**

### 6.2 调用链路

```
前端请求 episodes
  → Tauri command: get_episodes(series_id)
    → 查 episodes 表
      → 返回列表（title 可能为空）
    → 后台异步触发 fetch_episode_metadata(series_id)
      → 读 series.tmdb_id
      → 如果为空 → 跳过
      → call TMDB /tv/{id}/season/{season_number}
      → 遍历返回的 episodes，按 episode_number 匹配本地 episodes 记录
      → 更新 title, still_path, still_url, overview 等
      → 异步下载 still 缩略图（批量并发，限 5 个）
      → 写回 DB
```

### 6.3 剧集编号匹配

TMDB 返回的 episode_number 和 mochi 扫描器提取的 episode_number 需要对齐。大多数情况下一致（S01E01 → episode_number=1），但存在不一致的可能：

- **特别篇**：TMDB 可能把 SP 标记为 episode_number=0 或独立的 specials season（season_number=0）
- **编号偏移**：部分动画在 Bangumi 和 TMDB 的集数起始编号不同（如 Bangumi 第 1 话对应 TMDB 第 0 话）

匹配策略：
1. 默认按 `season_number + episode_number` 精确匹配
2. 如果匹配失败（TMDB 返回的某集在本地没有对应 episode），静默跳过该集
3. 如果本地有多集但 TMDB 返回的少（如还有未播出的），多的集保持 title 为空

---

## 7. 缩略图缓存

### 7.1 存储路径

沿用现有缓存规范：

```
{cache_dir}/cast/still_{tmdb_episode_id}.jpg
```

用 TMDB episode ID（不是本地 episode ID），跨 series 共享的同一剧集不会重复下载。

### 7.2 下载策略

- 尺寸：`w300`（剧集缩略图展示尺寸小，不需要大图）
- 并发：最多 5 个同时下载
- 失败处理：静默降级，still_path 留空
- 更新判断：对比 `still_url`，TMDB 换了图就重新下载

## 8. 前端展示（已确认）

### 8.1 详情页剧集区

两层结构：横滚 strip（主视觉）+ 「查看全部剧集」按钮（打开 modal）。

**横滚 strip**：
- 卡片 180×101（16:9），缩略图 `w300`，`border-radius: 6px`
- 集数编号覆盖左上角（半透明黑底白字，10px）
- 剧集名在下方 12.5px，单行截断
- 卡片间距 14px，横滚 + JS 软 snap（复用海报墙交互）
- 已看：右下角绿色小勾
- 续播位置：弱金色 `outline` 环（`opacity: 0.5`），与选中态强 accent 区分
- 进入详情页时自动滚到续播集左侧 1/3 处（`ease-out` ~400ms，入场动画完成后触发）
- 右侧边缘渐变遮罩
- 缩略图加载期间：暖金色 BreathingDot 占位（同海报占位逻辑）

**剧集 Modal**（居中弹出）：
- 680px 宽，`scale(0.94→1)` spring 入场，`max-height: 78vh`
- 3 列缩略图网格，gutter 16px
- 每张卡片：16:9 缩略图 + 集号覆盖 + 标题 + 时长
- 顶部：「全部 / 未看」筛选切换
- 已看卡片整体 muted（opacity 0.5）
- 续播卡片暖金 accent 环 + 「继续观看」文字
- 关闭：✕ / 遮罩 / ESC

### 8.2 缩略图加载策略

本地缓存优先。路由切换到详情页时立即发起下载（不等入场动画结束）。

- 首批 5 并发：~400-600ms（与 stagger 入场动画并行）
- 可见区域优先：strip 首屏 3-5 张先下载
- 网络慢（>1.5s）：骨架占位（暖金色 BreathingDot 脉动圆角矩形）→ 缩略图 fadeIn 替换
- 快速进出：AbortController 取消飞行中的请求
- 重新进入同一系列：本地缓存 0ms 加载

### 8.3 电影（仅 1 集）

不显示 strip。显示一行播放入口：剧集名 + 时长，点击直接播放。

## 10. 待讨论项（已全部决策）

1. **anime 的 TMDB ID 获取时机** → 元数据拉取时异步补，不阻塞扫描。
2. **TMDB 上找不到的动画** → title 留空，缩略图用暖金色纯色占位 + 集数编号。
3. **episode overview 要存吗** → 先存着，前端后面再决定用不用。
4. **剧集缩略图展示尺寸** → strip 用 `w300`（渲染 180px），modal 也用 `w300`（渲染 ~200px）。
