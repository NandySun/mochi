# mochi v0.2.3 更新记录

## 概述

本版本聚焦三个方向：**元数据刷新一致性修复**、**老番文件夹名称智能清洗**、**放映体验优化**。

---

## 一、元数据刷新一致性修复

### 1.1 详情页刷新与批量拉取行为统一

**问题**：详情页「刷新元数据」按钮调用链与批量拉取不一致，导致详情页刷新后系列标题/简介未更新。

**根因**：
- 详情页的 `fetch_metadata` → `fetch_cast` → `fetch_episode_metadata` 各自为独立 Tauri 命令，`fetch_cast` 和 `fetch_episode_metadata` 不接受 proxy 参数，通过 env var 读代理（为空）
- 批量拉取直接调用 metadata 模块函数，proxy 一路透传

**修复**：新增 `refresh_single_series` 命令，逻辑 1:1 对齐批量 per-series 处理。Phase 1 读 DB → Phase 2 `fetch_series_metadata`(force=true) → Phase 3 persist → Phase 4 `fetch_cast` + `fetch_episode_metadata`（直调模块函数，带 proxy）。

**涉及文件**：`commands.rs`、`lib.rs`、`SeriesDetail.tsx`

### 1.2 Anime cast 的 TMDB 兜底

**问题**：BLACK LAGOON 等动漫因 Bangumi 搜索失败（无 bangumi_id），`fetch_cast` 仅支持 Bangumi 角色获取，直接返回错误被静默吞掉，导致无演员表。

**修复**：`fetch_cast` 的 anime 分支增加三级兜底：
1. 有 bangumi_id → 试 Bangumi 角色
2. Bangumi 失败但有 tmdb_id → 兜底 TMDB credits
3. 无 bangumi_id 但有 tmdb_id → 直接用 TMDB

**涉及文件**：`metadata/cast.rs`

### 1.3 TMDB 搜索动画倾向过滤

**问题**：Cowboy Bebop 等在 TMDB 存在真人版的情况，TMDB 搜索返回真人版排在动画版前面，导致匹配到错误条目。

**修复**：`try_tmdb` 新增 `prefer_animation` 参数。为 `true` 时优先取 `genre_ids` 含 16（Animation）的结果。仅在动漫 Bangumi 搜索失败后的 TMDB 兜底路径启用。

**涉及文件**：`metadata/mod.rs`

### 1.4 剧集缩略图 force 重下载

**问题**：上一个问题修复后，系列元数据正确了但剧集缩略图仍是旧缓存的错误图片。`fetch_episode_metadata` 下载缩略图时 `force: false`，缓存存在即跳过。

**修复**：`fetch_episode_metadata` 新增 `force` 参数，`refresh_single_series` 和 `batch_fetch_all_metadata` 调用处传入 `true`，确保刷新时强制重新下载缩略图。

**涉及文件**：`metadata/episodes.rs`、`commands.rs`

## 二、老番文件夹名称智能清洗

### 2.1 `clean_anime_folder_name` 函数

**问题**：动漫 BDRip 文件夹命名常包含发布组标签、编码元数据、集数范围等噪音（如 `[Moozzi2] Samurai Champloo BD-BOX - TV + SP`），导致 `search_term` 无法命中 Bangumi/TMDB。

**修复**：新增 `clean_anime_folder_name`，两步清洗：

**Step 1 — 分类剥离方括号标签**：
- 发布组标签（首个括号、纯 ASCII ≤ 20 字符、不含 CJK）→ 丢弃
- 元数据标签（含分辨率/编码/音轨/字幕/封装关键词）→ 丢弃
- 集数范围标签（`01~26`、`01-16TV全集` 等）→ 丢弃
- 标题标签（以上都不是）→ 保留内容

**Step 2 — 清洗非括号残留**：
- 移除 `BD-BOX`、`- TV`、`+ SP`、`+ Movie`、集数范围、`Subs` 后缀等常见噪音

**集成**：在 `parse_folder_name` 之前调用，三个扫描路径（类型提示容器、传统扁平、文件聚合）均已接入。

**涉及文件**：`scanner.rs`

### 2.2 E 开头剧集文件名识别

**问题**：文件名 `E01上.mp4` / `E11a.mp4` 不匹配现有 8 个剧集正则模式，全部走 fallback 顺序编号。导致 E11a 被编号为 episode 3，TMDB 误匹配为第 2 期内容。

**修复**：ep_patterns 模式 1 从 `_E(\d{1,3})` 扩展为 `(?i)(?:^|_)E(\d{1,3})([上下ab])?`。支持识别文件名以 E 开头、后可跟 上下/ab 后缀的格式。后缀触发半集换算：上/a → 2N−1，下/b → 2N。（国产综艺一期分上下集的发布惯例）

**涉及文件**：`scanner.rs`

### 2.3 多作品合集文件夹自动拆分

**问题**：BLACK LAGOON 等老番 BDRip 常在一个父文件夹下放置多个独立作品（TV S1、TV S2、OVA），扫描器将其合并为一个系列。

**修复**：扫描器在类型容器路径和传统扁平路径中增加拆分逻辑：

- 检测文件夹深度 1 是否有视频文件（`has_video_at_depth1`）
- 无视频但深度 1 有子目录 → 每个子目录作为独立系列扫描
- 有视频 → 保持原行为（单系列）

辅助：抽取 `build_series_scan` 公共函数消除重复代码。

**涉及文件**：`scanner.rs`

## 三、放映体验优化

### 3.1 全屏切集不退出

**问题**：播放器全屏状态下切换到下一集，组件重新挂载导致全屏 React state 重置为 false。

**修复**：
- Rust 侧新增 `get_fullscreen` 命令，读取原生窗口真实全屏状态
- `useFullscreen` hook 挂载时调用 `get_fullscreen`，不再盲目初始化 `false`

**涉及文件**：`commands.rs`、`lib.rs`、`hooks/useFullscreen.ts`

### 3.2 双击全屏

**新增**：视频区域（`data-mpv-area`）双击切换全屏/退出全屏。不干扰 OSC 控件点击。

**涉及文件**：`VideoPlayer.tsx`

## 四、详情页操作增强

### 4.1 单系列重新扫描

**新增**：详情页 kebab menu 新增「↺ 扫描新剧集」按钮。针对追更场景，无需全库扫描即可检测单文件夹新增/移除的剧集。

**实现**：`rescan_series_folder` 命令，在注册的 root_path 中定位文件夹 → `scan_series_folder` → upsert 剧集 → 清理已删除剧集。支持层级（`root/type/folder`）和扁平（`root/folder`）两种布局。

**涉及文件**：`commands.rs`、`db.rs`、`scanner.rs`、`lib.rs`、`SeriesDetail.tsx`
