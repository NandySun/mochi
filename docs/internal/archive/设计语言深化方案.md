# Mochi 设计语言深化方案

> 2026-06-21 · 纯设计讨论产出，待新对话实施

---

## 零、核心原则：材质物理三定律

mochi 设计语言不是一组零散风格的集合，而是从「麻糬」这个意象推导出的三条底层物理规则。所有交互决策以此校验。

**定律 1：形变可逆。**
任何被按压、拉伸、挤压的元素，最终都回归原始形态。变形是暂时的响应，不是永久的状态。

**定律 2：运动有质量。**
没有东西瞬间出现或消失。所有运动携带惯性——启动有微小延迟，停止有回弹。感知窗口控制在 80-150ms，存在但不觉迟缓。

**定律 3：温度是信息。**
颜色、光影的变化承载叙事。冷色 = 距离/思考/等待，暖色 = 亲近/完成/在场。模糊仅服务于可读性，不作为装饰性转场语言。

## 一、加载态：呼吸的圆

### 1.1 设计决策

确立一个唯一的加载态符号：**轻轻呼吸的圆**。微微膨胀收缩，像一块麻糬在呼吸。

不做多个加载态变体。不做弹性进度条、脉冲骨架屏、扫描进度浮层。不同场景下**符号不变，变的是符号所在上下文**。

### 1.2 场景分级

| 场景 | 符号规格 | 上下文 |
|---|---|---|
| 海报图懒加载（<2s） | 16px，暖米色 | 出现在海报占位格中央；海报加载完成后圆 fade out，海报 fade in |
| 元数据拉取（3-15s） | 24px，暖米色 | 出现在触发位置旁（如详情页标题下方），像在静静等待的小东西 |
| 全库扫描（>15s） | 48px，暖米色 | 圆不变，下配一行缓慢更新的扫描路径文字 + 扫描计数。圆的呼吸和文字的更新节奏独立 |
| 短操作（<2s） | 16px，暖米色 | 如切换筛选标签后的列表刷新；出现在内容区中央 |

### 1.3 动画参数

- 呼吸周期：约 2s 完成一次膨胀 → 收缩 → 膨胀
- 缩放幅度：scale 1.0 ↔ 1.15，ease-in-out
- 不做旋转，不做位移，只有呼吸

### 1.4 反模式

不得出现转圈 spinner。转圈是机械隐喻，和 mochi 的材质语言冲突。

## 二、果冻标签指示器：scaleX 微调

### 2.1 设计意图

当前 `layoutId="filter-indicator"` 的 spring morph 已传递「这个指示器是软的」的感知。在其上叠加沿移动方向的 scaleX 变形，将感知推进到「软的东西被拖拽时会拉长」——这是同一套物理直觉的延伸，不是新语言。

判断标准：如果一块真正的麻糬会这样动，就可以做。麻糬被移动时会沿移动方向拉伸——所以可以做。

### 2.2 动画规格

- 正向拉伸：scaleX 1.06~1.10（沿移动方向）
- 反向回弹：scaleX 0.94~0.97（短暂 undershoot）
- 总时长：280~350ms，与 layoutId spring duration 大致对齐
- transform-origin：运动方向的反侧（让拉伸看起来是「向前探」而非「原地膨胀」）

### 2.3 实现方向

保留现有 `layoutId` 管理位置和宽度。在指示器内部的视觉元素上挂 `useMotionValue(scaleX)`，通过 `useEffect` 监听选中 tab 变化方向触发一次 keyframe 动画。两个动画轨道作用在不同维度，不直接冲突。

### 2.4 幅度警戒

- scaleX > 1.2 → 橡皮筋感，脱离安静气质，不可取
- scaleX < 0.85 → 弹弓感，同上
- 目标水位：用户注意不到它，但砍掉后会觉得「好像没有之前顺滑了」

## 三、按压形变系统：squish

### 3.1 当前状态

`whileHover` + `whileTap` 已覆盖海报卡片和筛选标签按钮。但播放器控件区仍有多个交互表面仅使用 CSS hover 变色。

### 3.2 扩展清单

| 交互节点 | 方案 |
|---|---|
| 播放/暂停按钮 | `whileTap scale=0.88` + spring bounce back，模拟按下实体按钮 |
| 进度条拖拽拇指 | 拖拽时 scale 1.15，释放时 spring 弹回 scale 1.0 |
| 音量滑块拇指 | 同上 |
| 设置页开关 | 自定义 toggle 替换原生 checkbox；切换时 pill 做弹性形变 |

### 3.3 设计意图

按压形变是 mochi 设计语言中最高频的触感词汇。每一次按压，视觉上的凹陷和弹回与手指在触控板上的物理反馈形成通感。这是「这个界面是可以摸的」最直接的表意。

## 四、滚动粘度

### 4.1 设计意图

moochi 的滚动不应模拟金属滑轨（iOS 惯性滚动），而应模拟搅拌稠糯米糊——有惯性，但被温柔的粘滞力包裹。

### 4.2 具体表现

- **停止回坐**：滚动 momentum 归零时，不做硬停止，做 3-5px 的 elastic settle（rubber-band）
- **snap 磁吸**：海报墙 snap 点之间有渐进吸引力——当 momentum 接近 snap 点（阈值约 30px），snap 点产生渐强吸力（spring tension 随距离减小而增大），不跳变
- **方向反转**：快速来回滚动时，内容有肉眼可见但不致迟滞的惯性延迟

### 4.3 实现基础

framer-motion 的 `useScroll` + `useTransform` 提供所需原语。当 CSS `scroll-snap-type: mandatory` 被替换为 JS 自定义 snap 逻辑时，上述行为可在一个统一的物理模型中实现。

## 五、颜色温度叙事

### 5.1 设计意图

系列间导航时，背景 crossfade 不仅换颜色，更传递体感温度变化。冷色 → 暖色和暖色 → 冷色的过渡曲线不对称，分别传达「步入温暖」和「温暖被抽走」的不同体感。

### 5.2 过渡规格

- 总时长：1.5s
- 冷 → 暖：冷色占 60% 过渡时间缓慢消退，最后 40% 暖色快速渗入
- 暖 → 冷：暖色在 30% 时间内快速消退，剩余 70% 冷色缓慢渗入

### 5.3 前置依赖

需提取每部作品海报的主色调（dominant color）。在海报首次加载时完成提取并缓存。成本极低。

## 六、空白状态叙事温度

### 6.1 设计意图

mochi 中的「空」不止一种，每种语义不同，视觉反馈不应相同。

### 6.2 分类

| 空状态类型 | 触发条件 | 视觉表现 |
|---|---|---|
| 全新空库 | 首次启动，无媒体文件 | 已有：呼吸动画 + 引导文字。暖光，期待感 |
| 筛选空结果 | 选中标签下无对应媒体 | mochi 形态微微低头（rotate -5°），不沮丧，更像是「换个标签试试？」 |
| 扫描中 | 全库扫描进行中 | 呼吸的圆 + 路径文字（见第一章加载态层级 3） |

## 七、已否决提案

### 7.1 光影叠层叙事（毛玻璃转场）

**提案：** 详情页切换到播放页时，详情页叠 `backdrop-filter: blur(12px)` + 半透明，播放器从毛玻璃「后面」浮现。

**否决理由：**

1. **语言冲突。** 模糊 = 距离感，是摄影/电影语言。mochi 的核心是触觉通感——按压、弹跳、呼吸——暗示「可以摸到」。引入距离感和亲近感矛盾。
2. **没有功能。** 筛选栏和 OSC 的毛玻璃服务于功能可读性（让文字浮在变化背景上）。叠层叙事中的毛玻璃不服务任何功能，纯粹是装饰性延迟。
3. **降低体验。** 在已工作的 slide 过渡中间插入 300ms 毛玻璃态，实际效果是多了 300ms 不可操作时间。

**判断标准复用：** 真正的麻糬不会变模糊——它没有焦平面。所以不做。

## 八、实施建议

### 优先级

1. **加载态呼吸的圆**（高优先）——填补当前空白，解决真实用户痛点（扫描和元数据拉取无反馈）
2. **果冻标签 scaleX**（中优先）——深化已有语言，代价低，不和现有逻辑冲突
3. **按压形变扩展**（中优先）——播放器控件区触感统一
4. **滚动粘度**（低优先）——需替换 CSS snap 为 JS 自定义，改动较大
5. **颜色温度叙事**（低优先）——依赖主色调提取前置工作
6. **空白叙事**（低优先）——筛选空结果目前可能不可达或极少触发

### 实施原则

- 每次只做一个方向，验证交互手感后再推进下一个
- 幅度从保守起步，不够再加，避免一步过头
- 每个方向实施后在实际设备上测试，不以模拟器/浏览器中的感受为准
