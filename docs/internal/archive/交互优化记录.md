# Mochi 交互优化记录

> 2026-06-21 · v0.1.2

---

## 概述

本次更新引入 framer-motion 11.x，对 mochi 全局交互动效进行系统化升级。覆盖页面过渡、海报墙、详情页、播放器 OSC、设置弹窗，并修复了引入过程中暴露的三个回归问题。

---

## 新增依赖

```json
"framer-motion": "^11.x"
```

选型理由：spring 物理动画原生支持、声明式手势（whileHover/whileTap）、AnimatePresence 路由过渡、layoutId 共享元素。桌面端不必担心 bundle size（~30KB gzip）。

---

## 全局架构变更

### 路由层

```tsx
// v0.1.1
<Routes>
  <Route ... />
</Routes>

// v0.1.2
<LayoutGroup>
  <AnimatePresence mode="wait">
    <Routes location={location} key={location.pathname}>
      <Route element={<PageWrapper config={...}>...</PageWrapper>} />
    </Routes>
  </AnimatePresence>
</LayoutGroup>
```

- `mode="wait"`：旧页面退出动画完成后再进入新页面
- `key={location.pathname}`：确保路由切换时 AnimatePresence 识别子节点变化
- `LayoutGroup`：为 Phase 5 共享元素过渡（layoutId 海报 morph）预留

### 导航方向感知

`useNavigationDirection` 维护 pathname 历史栈，在 render 阶段同步推导方向。前进用 `spring.page`（慢柔惯性），后退用 `spring.back`（快 ~20%）。

### PageWrapper animate 推导

从 `initial` 自动推导 `animateTarget`——只设 initial 中存在的属性。这避免了在不需要 transform 的页面（特别是播放页）上留下 identity transform（如 `translateY(0px)`），防止 Chromium 透明窗口下 GPU 合成层遮挡 mpv 原生表面。

## 回归问题与修复

### 问题 1：首页滚轮切换失效

**现象**：滚轮触发 selectedIndex 变化，但海报不跟随居中。

**根因**：`animate(container, { scrollLeft: target }, spring.gentle)` —— framer-motion 的 `animate()` 命令式 API 对 `scrollLeft`（DOM 属性，非 CSS 属性）的 WAAPI/JS 回退路径静默失效。

**修复**：回退为原生 `container.scrollTo({ left: target, behavior: "smooth" })`。

### 问题 2：播放器有声音无画面

**现象**：视频可播放、有音频，但画面全黑。

**根因**：Phase 1 在 `index.html` 防白屏 `<style>` 中写了 `html, body, #root { background: #0a0a0a }`。`#root` 是 React 组件树的容器，覆盖整个视口。其 `#0a0a0a` 背景在播放页不可被 VideoPlayer 的 useEffect 覆盖（因为 `#root` 不在该 effect 的作用范围内），直接遮挡了下层 mpv 原生表面。v0.1.1 中 `#root` 无背景色，链路通畅。

**修复**：防白屏 style 中去掉 `#root`，仅保留 `html, body`。

**其他排查过程中排除的疑似原因**：
- PageWrapper 的 identity transform 残留（已修复：从 initial 推导 animateTarget，播放页无 transform）
- `will-change: transform` CSS 规则（已移除，替换为注释说明）
- GlobalBackground parallax 合成层（播放页 fanart 为 null，不渲染）

### 问题 3：进度条端点留在 0 秒

**现象**：黄色进度条主体正常推进，白色端点圆点始终停留在起始位置。

**根因**：`useSpring(currentPct)` + `useTransform` + `MotionValue` 传入 `motion.div` style 的管线，在 `currentPct` 高频变化（mpv 每 ~100ms 回调）时，render/effect 时序下 MotionValue 无法正确传播到 DOM。进度条填充走纯 React render 路径（`width: ${currentPct}%`）不受影响，形成"条动点不动"的分裂。

**修复**：端点回退为普通 `<div>` + `style={{ left: \`${currentPct}%\` }}`，与进度条填充走相同路径。

### 问题 4：启动白屏闪烁

**现象**：启动时可见 白色→黑色→界面 的三段过渡。

**根因**：Tauri 窗口创建到 WebView 首帧之间，Windows DWM 用系统默认白色填充透明窗口。

**修复**：`tauri.conf.json` 窗口配置加 `"backgroundColor": "#0a0a0a"`，作为首帧前的 fallback 色。

# Mochi 交互优化记录

> 2026-06-21 · v0.1.3

---

## 概述

本次更新聚焦 Bug 修复和交互打磨：修复自动连播穿刺、音量按钮交互、窗口缩放失效，并为首页筛选栏引入果冻切换动效。窗口缩放问题的排查过程中，深入探索了 Tauri v2 + WebView2 的窗口层级和 Tao 内置 resize 机制。

---

## Bug 修复

### Bug 1：详情页点击选集 → 自动连播穿刺到观看记录

**现象**：在 SeriesDetail 点击任意剧集，播放器先加载该集，随后一集一集自动往后跳，直到停在有观看记录的剧集。

**根因**：`VideoPlayer` 自动连播逻辑不区分"用户自然看到结尾"和"刚加载就被 resume 到接近结尾"。resume 跳转到接近结尾的位置会立即触发自动连播条件（`timePos >= duration - 3`），形成穿刺链条。

**修复**：新增 `hasPlayedNaturallyRef` 守卫。playhead 进入过 0~5 秒区间才翻为 `true`，自动连播增加前置条件 `hasPlayedNaturallyRef.current === true`。resume 跳转从不经过低区间，被自然拦截。

**文件**：`src/components/VideoPlayer.tsx`

### Bug 2：OSC 音量按钮交互

**2a. Hover 触发太敏感**：鼠标扫过音量按钮即弹出滑块，与轨道菜单（⋮）和选集浮层（📋）的 click to toggle 模式不一致。

**修复**：音量浮层从 hover 驱动改为 click toggle，新增 click-outside 关闭逻辑，统一交互模式。

**2b. 滑块水平居中偏移**：音量浮层的 `motion.div` 同时承载 framer-motion 的 `y` 动画和内联 `transform: translateX(-50%)` 水平居中。framer-motion 管理自己的 transform，覆盖手写的 `translateX(-50%)`，导致滑块偏位。

**修复**：定位与动画解耦。外层静态 `<div>` 负责 `translateX(-50%)` 水平居中，内层 `motion.div` 只负责入场/退场动画。

**文件**：`src/components/OscBar.tsx`

### Bug 3：无法拖拽边框缩放窗口

**现象**：鼠标移到窗口边缘不出现 resize 光标，无法拖拽调整窗口大小。

**排查过程**：
1. **Rust 侧 WM_NCHITTEST 子类化**：尝试在主窗口和子窗口上拦截 `WM_NCHITTEST` 返回 HT* 值。发现主窗口根本收不到 `WM_NCHITTEST`（被 WebView2 子窗口拦截）。
2. **子类化子窗口**：通过 `EnumChildWindows` 枚举完整窗口层级，发现 `TAURI_DRAG_RESIZE_BORDERS`（Tao 内置）、`WRY_WEBVIEW`、`Chrome_WidgetWin_*`、`Chrome_RenderWidgetHostHWND` 等深层窗口。最深层的渲染面 `Chrome_RenderWidgetHostHWND` 不允许子类化（COM 安全限制，`SetWindowLongPtrW` 返回 0）。
3. **Tao 内置机制**：`TAURI_DRAG_RESIZE_BORDERS` 窗口存在但不工作——排除 `WS_EX_LAYERED` 和 `dragDropEnabled` 配置干扰后仍无效，判断为 Tao v0.35.3 的版本 bug。
4. **Tauri v2 权限系统**：前端 `setSize`/`setPosition` 调用静默失败，根因是 `capabilities/default.json` 未开放窗口操作权限（`core:window:allow-set-size` 等）。
5. **IPC 延迟导致首帧跳变**：`onPointerDown` 中等待 `innerSize()`/`outerPosition()` 异步 IPC 返回（~100ms），期间用户已移动鼠标。回调激活时基准坐标是旧值，首帧 `dx` 达上千像素。

**修复**：
- **前端 resize handles**（`src/components/WindowResizeHandles.tsx`）：在所有窗口边缘渲染 5px 透明 handle（四角 15px），`setPointerCapture` 捕获拖拽事件，`requestAnimationFrame` 节流 IPC 调用。
- **同步激活**：用 `window.innerWidth/innerHeight`（同步 DOM 值）替代 `innerSize()` IPC 取尺寸，`dragging.current` 在 `pointerdown` 时即时激活，消除首帧跳变。
- **增量 IPC**：同一轴上坐标不变时跳过 `setPosition`，仅调必要的 API。
- **权限开放**：`capabilities/default.json` 增加 `set-size`、`set-position`、`unmaximize`、`inner-size`、`outer-position` 等窗口权限。
- **配置清理**：`tauri.conf.json` 移除废弃的 `dragDropEnabled`（v1 残留）。
- **Rust 代码回退**：`lib.rs` 和 `Cargo.toml` 回归原始状态，无 WM_NCHITTEST 子类化代码，无 `windows` crate 依赖。

**文件**：
- 新建 `src/components/WindowResizeHandles.tsx`
- 修改 `src/App.tsx`
- 修改 `src-tauri/capabilities/default.json`
- 修改 `src-tauri/tauri.conf.json`
