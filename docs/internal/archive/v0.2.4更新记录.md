# mochi v0.2.4 更新记录

## 概述

本版本新增 Kodi `.nfo` 文件解析支持。从 Jellyfin / Kodi 迁移过来的系列文件夹中如有现存 NFO 文件，mochi 扫描时将自动读取其中的描述性元数据，避免重复刮削。

---

## 一、NFO 元数据导入

### 功能

扫描时自动检测系列文件夹内的 `tvshow.nfo` 或 `movie.nfo`（Kodi XML 格式），解析以下字段：

| NFO 字段 | 导入目标 |
|---------|---------|
| `<plot>` | 系列简介 |
| `<year>` / `<premiered>` | 年份 |
| 多个 `<genre>` | 类型标签 |

**涉及文件**：`nfo.rs`（新增）、`db.rs`、`commands.rs`、`lib.rs`、`Cargo.toml`

### 设计原则

**被动兼容，不增加开关。** 没有 NFO 文件 = 什么都没有发生。有 NFO 文件 = 元数据自动填入。用户不需要感知功能的存在。

**不覆盖在线数据。** NFO 元数据优先级低于 Bangumi / TMDB 拉取结果。如果用户后续在线拉取了元数据，NFO 数据会被覆盖；如果先在线拉取后重新扫描，NFO 不会覆写已有数据。

**独立模块，不侵入扫描器。** NFO 解析在扫描完成后以独立后处理步骤运行，与 `scan_series_folder` 互不耦合。解析失败静默跳过，不影响扫描流程。

### 不做的事

- 剧集级 NFO（`E01.nfo`）暂不解析
- `<fileinfo>` 技术参数（codec、duration 等）忽略
- `<actor>` 块忽略（无头像 URL 数据不对齐 Person 表）
- mochi 不写入 NFO 文件（只读）
