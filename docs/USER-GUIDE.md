# Mochi 使用指南

> 适用版本：v0.3.5（2026-06-30）

这份文档面向使用 Mochi 管理本地影库的人。如果你是第一次打开 Mochi，从 [快速入门](#快速入门) 开始；如果你卡在某个具体问题，直接跳到 [Q&A](#qa)。

---

## 快速入门

Mochi 启动后第一件事是判断「媒体库在哪儿」。

**首次启动**

1. 看到引导页。两个选项：
   - **创建推荐文件夹结构** — 在你选的目录下创建 `anime/`、`movie/`、`tv/`、`variety/` 四个子文件夹。适合从零开始
   - **直接进入设置** — 不创建任何文件夹，直接跳到设置页。适合你已经有现成的媒体库
2. 无论哪个选项，下一步都是相同的：**添加媒体库根目录**。
   - 设置 → 媒体库 → 添加文件夹
   - 选择要纳入管理的根目录（如 `D:\Media`）
   - 弹出窗口中指定这个根目录下的内容属于哪种类型
3. 回到媒体库页，点 **重新扫描**。Mochi 会递归扫描根目录，识别视频文件、按文件夹分组、推断季号
4. （可选）填入 TMDB API Key。设置 → 媒体库 → TMDB API Key。动漫不需 Key，影视需要
5. 批量拉取元数据。设置 → 媒体库 → 批量拉取。完成后回到首页

**关于 TMDB Key**

TMDB 是免费的影视数据库。注册地址：themoviedb.org/signup。申请 API Key 选「Developer」类型，个人使用完全免费。Mochi 只在拉取影视类元数据时使用它，Key 存在本地，不上传任何服务。

**关于网络**

- Bangumi（动漫）在国内可能不可达，需要代理
- TMDB 在国内偶有抽风，但不需代理也能用
- 一次拉取的元数据会缓存到本地，断网也能翻海报

## 媒体组织

### 三种类型标记方式

Mochi 不强求特定结构。它通过三种方式识别一个文件夹的类型：

| 方式 | 示例 | 何时用 |
|---|---|---|
| **添加时指定** | 设置 → 添加文件夹 → 选「动漫」 | 文件夹已经存在且不想改名 |
| **文件夹名后缀** | `进击的巨人 [anime]`、`浴血黑帮 [tv]` | 希望文件夹名自描述，看一眼就知道 |
| **子目录约定** | `D:\Media\anime\进击的巨人\` | 从零开始，习惯分类管理 |

三种方式可以混用。优先级从高到低：

1. `.mochi` 文件（手动标记的最高优先）
2. 文件夹后缀 `进击的巨人 [anime]`
3. 父目录名（`anime/`、`tv/`、`movie/`、`variety/` 这种约定）
4. 添加根目录时指定的类型（对该根目录下所有顶层系列生效）
5. 全部未命中 → `unknown`（元数据拉取时让 API 推断）

**`anime/`、`movie/`、`tv/`、`variety/`** 这四个目录名是约定俗成的「父目录类型标记」。如果你把视频放在 `D:\Media\anime\` 下的任意子文件夹里，整个 `anime/` 子树都会被当作动漫处理。

### 嵌套布局 vs 扁平布局

- **嵌套布局（默认）** — `根目录/类型/系列名/Season X/EP01.mkv`。每季一个文件夹。适合从零整理
- **扁平布局** — `根目录/系列名/EP01.mkv`，没有季文件夹。适合已经下载好的单季番

切换方式：根目录添加时选择模式。两种模式可以并存，每个根目录独立配置。

### 季号识别

扫描器会从文件夹名自动提取季号，应用到该文件夹下的所有剧集：

- 中文：`第六季` / `Season 6` / `S6`（含「六」「七」等中文数字）
- 英文：`Season 6` / `S6`
- 简写：仅匹配末尾的 `S6`

`S06E01` 这种文件名里直接写的季号会优先于文件夹名。

**例子**：`D:\Media\anime\进击的巨人\第三季\EP01.mkv` → 季号 = 3

### 手动覆盖

在详情页的 **类型选择器** 里可以随时改一个系列的类型。系统会尊重你的选择（v0.3.5 修复了之前类型被 Bangumi 通道覆盖的问题）。

## 元数据

### 双搜策略

| 类型 | 主搜 | 横幅 | 说明 |
|---|---|---|---|
| 动漫 (anime) | Bangumi | TMDB 补拉 | Bangumi 是中文动漫最准确的来源 |
| 影视 (tv / movie) | TMDB | TMDB | 全球影视数据 |
| 综艺 (variety) | TMDB | TMDB | Bangumi 不覆盖综艺，必须走 TMDB |
| 未知 (unknown) | 并行 | — | 系统会先尝试 TMDB，失败再尝试 Bangumi |

### 评分

Mochi 显示的评分是 **TMDB 0–10 分制** 和 **Bangumi 0–10 分制**。

- 7.4 在 TMDB 是相当不错的分数（接近 IMDb 7.4 的口碑水平）
- Bangumi 上 8.0 以上的番就算高质量
- 如果你看到 8.8+ 的高分布局，多半是经典

不要按国内某些网站的 0–100 习惯来理解。

### 手动匹配

自动匹配不准确时（在详情页 ⋮ 菜单 → 编辑搜索词），可以：

1. 修改搜索词（通常是中文名 / 英文名 / 年份组合）
2. 在结果列表里手动指定 ID

手动匹配成功后系统会记住 ID，下次刷新元数据不会再走自动搜索。

### 批量拉取

设置 → 媒体库 → 批量拉取元数据。

- 后台运行，关闭设置页不中断
- 进度条显示 `{已完成}/{总数}`
- 完成后顶部 banner 提示
- 失败时可在设置 → 媒体库看到失败项

**性能建议**：网络稳定时一次性拉取几百部没问题。带宽有限时分批（按类型筛选后再批量）。

### 文件夹季号识别

扫描器在 `[type]` 后缀、季号识别、文件夹名清洗（剥离发布组标签、编码元数据、集数范围）上做了不少工作。常见情况：

- `进击的巨人 (Shingeki no Kyojin) [BD 1080p]` → 识别为 `进击的巨人`，动漫类型
- `The.Mandalorian.S03.1080p.WEB-DL` → 季号 = 3
- `七龙珠.第01-100话.BDrip` → 老番文件夹，扫描器会自动展开范围

如果你的命名规范很独特，识别可能不准。可以在详情页用 ⋮ → 编辑搜索词手动修正。

## 浏览

### 筛选栏

顶部六个 tab：

- **全部** — 所有系列
- **继续** — 有未看完剧集（不是 100% 进度的系列）
- **动漫 / 影视 / 电影 / 综艺** — 按类型

### 全局搜索

在首页按 `/` 或 `Ctrl+K` 唤起居中搜索浮层。

- 搜索范围：系列 title、搜索词、年份
- 60 ms 防抖，输入即查
- 最多 8 条结果，↑↓ 导航，Enter 进入详情
- ESC 关闭

### Indicator Dots

海报墙下方的小圆点表示系列位置。圆点可点击直跳，超过 7 部时窗口化显示（左右各渐隐）。同时支持键盘 ← → 切换。

## 详情页

### 杂志跨页布局

详情页是固定三段式结构：

1. **顶部海报 + 元数据** — 海报在左，右侧从上到下是标题、评分、年份、集数、类型选择器、操作按钮、⋮ 菜单
2. **演员表** — 横向滚动圆形头像
3. **剧集 strip** — 横滚缩略图卡片，续播集金色高亮

页面**不展开**完整简介和完整剧集列表——这两个都通过弹窗承载（ⓘ 简介触发点 / 查看全部剧集按钮），保持页面紧凑。

### 操作

- **继续** / **从头开始** — 顶部按钮，按你的状态切换
- **类型选择器** — 改了之后会触发重新拉元数据
- **⋮ 菜单**（按显示顺序）：
  1. ↻ 刷新元数据
  2. ↺ 扫描新剧集（追更时拉新文件）
  3. ✎ 编辑搜索词
  4. ↓ 导出 NFO（已导出过变「↓ 重新导出 NFO」）
  5. ✕ 清除 NFO

## 播放

### 快捷键

| 键 | 功能 |
|---|---|
| `Space` | 播放 / 暂停 |
| `←` `→` | 后退 / 前进 5 秒 |
| `↑` `↓` | 音量 |
| `F` | 全屏切换 |
| `Esc` | 退出全屏 / 返回 |
| `M` | 静音 |
| `[` `]` | 字幕延迟 ± 0.1 秒 |
| `P` | 上一集 |
| `N` | 下一集 |
| `J` | 循环字幕轨道 |
| `K` | 循环音轨 |

完整列表在设置 → 快捷键。

### OSC 主题

五种预设：Mochi（暖金，默认）/ YouTube / PotPlayer / Netflix / 极简。设置 → 播放 → OSC 主题切换。

### 进度保存

- 每 10 秒自动保存
- 切集时保存
- 关闭窗口时保存
- 进度达到 95% 标记为「已看完」

### 自动连播

播放到剧集末尾倒数 3 秒自动跳转下一集。**当前版本没有关闭选项**——一旦开始下一集会自动连播。如果想停在某集：剧集结束前手动暂停，或在播放页直接返回上一级退出播放器。

### 全屏

`F` 键切换。全屏时顶栏 + OSC 3 秒无操作自动淡出，鼠标移动重新显示。

### 音轨 / 字幕切换

OSC 右侧 **📋** 按钮打开选集浮层。音轨和字幕在浮层里切换。v0.3.4 起切换体验稳定，之前的「音轨切换不响应」问题已修复。

## 字幕与字体

### 异源外挂字幕自动识别

视频和字幕来自不同发布组时（例如视频是 A 组，字幕是 B 组），文件名无共同前缀。Mochi 会回退到 **集号正则匹配**，找到对应字幕文件加载。

支持的字幕格式：`.ass` / `.ssa` / `.srt` / `.sub` / `.vtt` / `.idx`

扫描位置：视频同目录、`sub/`、`subs/` 子目录。

### 系列级字体自动加载

如果你的 ASS 字幕里用了 `{\fn方正兰亭黑_GBK}` 这种字体标签，但系统没装这个字体，可以把字体文件放进系列文件夹下的 `fonts/` 目录：

```
进击的巨人/
├── fonts/
│   ├── 方正兰亭黑_GBK.TTF
│   └── 思源黑体.OTF
├── Season 1/
│   ├── EP01.mkv
│   └── EP01.ass
```

- 识别大小写不敏感：`fonts` / `Fonts` / `FONTS` 都行
- 没有 `fonts/` 目录完全不影响播放（零回退风险）
- 仅识别**目录**，`Fonts.rar` 这种压缩文件不算
- 单集外挂字幕（视频同级）走原有路径，不受这个影响

### 关于 `api.bgm.tv` 不可达

Bangumi API（`api.bgm.tv`）在国内通常需要代理。如果你看到动漫搜索一直失败但 TMDB 正常，多半是网络问题。Mochi 没有内置代理设置，需要在系统层面配置。

## NFO

NFO 是 Kodi 时代的元数据交换格式。Mochi 双向支持：读取已有 NFO 来填补扫描数据，也把自己的元数据写回 NFO，让 Emby / Jellyfin / Kodi 这些「邻居」能直接读取。

### 读取 NFO（自动）

扫描时自动检测系列文件夹里的 `tvshow.nfo` / `movie.nfo`，解析 `<plot>`、`<year>`、`<genre>` 等字段。

- **优先级最低** — 在线拉取的元数据优先于 NFO
- **零开关** — 没有 NFO 时零开销
- **不写集级** — mochi 集级数据稀薄，只在系列级 NFO 写元数据

### 写回 NFO

触发方式：

- **单系列** — 详情页 ⋮ → 导出 NFO
- **批量** — 设置 → 媒体库 → 批量导出 NFO

写回内容包括：title / plot / year / genre / uniqueid (tmdb + bangumi)。同时复制 `poster.jpg` 和 `fanart.jpg` 作为 sidecar（**仅当文件夹里没有同名文件时**——你手动放的优先）。

**覆盖保护**：

- 默认不覆盖：目标 NFO 已存在时返回错误，UI 弹窗提示
- 用户在弹窗里点「覆盖导出」才会真的覆盖
- 批量导出时，已存在的 NFO 计为「跳过」

**用途**：从 mochi 迁移到 Jellyfin / Emby / Kodi，或者把元数据分享给其他工具。

### 清除 NFO

触发方式：

- **单系列** — 详情页 ⋮ → 清除 NFO
- **批量** — 设置 → 媒体库 → 清除所有 NFO

**警告**：清除 NFO 时可选删除 `poster.jpg` / `fanart.jpg`。Mochi **无法区分**自己写的 sidecar 和你手动放的——勾选「包括 sidecar」会一视同仁删除。

**幂等**：文件不存在不报错。批量调用无需 per-file 守卫。

## 设置

6 个标签页（按侧边栏顺序）：

| 标签 | 内容 |
|---|---|
| 媒体库 | 根目录管理（含类型标记）、批量 NFO 操作、元数据源（TMDB Key / 代理 / 批量拉取） |
| 播放 | OSC 主题选择器 |
| 通用 | 关闭窗口行为（托盘 / 退出）、语言（占位） |
| 快捷键 | 按页面分组列出全部快捷键 |
| 数据 | 图片缓存（大小 + 清除）、元数据重置、观看记录清除、恢复出厂设置 |
| 关于 | 动态版本号（来自 `tauri.conf.json`）、技术栈、致谢、许可证、检查更新 |

### 关闭行为

- **最小化到托盘** — 关闭按钮最小化到系统托盘，托盘右键菜单可以退出
- **彻底退出** — 关闭按钮直接退出进程

托盘右键「退出」始终真正退出，不受关闭行为设置影响。

### 缓存

- 位置：安装版在 `%APPDATA%/mochi/cache/`，便携版在 exe 同级 `cache/`
- 内容：海报、剧照、演员头像
- 管理：设置 → 数据 → 图片缓存。不会清除 NFO

## 数据与便携模式

### 数据位置

数据根目录由 exe 同级的 `.portable` 标记文件控制：

- **存在 `.portable`** — 便携模式，所有数据（DB / config / cache）存 exe 旁边
- **不存在** — 安装模式，数据存 `%APPDATA%/mochi/`

### 双轨分发

- **安装版** — `Mochi_0.3.5_x64-setup.exe`，NSIS 安装器，自动创建开始菜单快捷方式
- **便携版** — `mochi-v0.3.5-portable.zip`，解压即用，U 盘 / 网盘随身带

两个版本从同一份代码打包出来，行为完全一致，只是数据位置不同。

### 数据迁移

把数据从安装版迁移到便携版（或反过来）：

1. 找到源数据目录（安装版：`%APPDATA%/mochi/`；便携版：exe 同级）
2. 整个目录复制到目标位置
3. 启动目标版本

数据库（`mochi.db`）和配置（`config.json`）在一起，缓存可以重新拉取。

## Q&A

**Q1. 评分显示 7.4，是不是匹配错了？**

不是。Mochi 显示的是 TMDB / Bangumi 的 0–10 分制，不是国内的 0–100。TMDB 7.4 相当于 IMDb 7.4 口碑水平，已经相当不错。

**Q2. 为什么综艺没匹配到结果？**

综艺（variety）只在 TMDB 里有数据，Bangumi 不覆盖。确保你填了 TMDB API Key，并且系列被标记为「综艺」类型。

**Q3. 怎么改一个系列的类型？**

详情页 → 类型选择器（标题行右侧下拉）。改完会触发重新拉元数据。

**Q4. 详情页「继续」按钮在哪？**

标题行右侧两个按钮的第一个。优先显示「继续 E05」（上次看到的集），看完后变「从头开始」。

**Q5. 自动连播能关吗？**

当前版本不能。Mochi 没有提供关闭自动连播的选项，剧集结束前 3 秒会自动跳到下一集。规避办法：剧集结束前手动暂停，或直接返回上一级退出播放器。

**Q6. 关闭后进度会丢吗？**

不会。进度每 10 秒 + 切集 + 关闭窗口时都会保存。回到首页「继续」tab 就能看到未看完的系列。

**Q7. 「已看完」的剧集还在「继续」分类里？**

「已看完」标记是 95% 以上进度。差一点没到的不算。如果一集看完自动连播到下一集，但下一集刚开始就关了，那这集还在「未看完」状态。

**Q8. NFO 写回会覆盖我手动写的吗？**

不会。默认遇到已有 NFO 直接返回错误，UI 弹窗提示「是否覆盖」。你在弹窗里点「覆盖导出」才会真覆盖。

**Q9. 清除 NFO 会不会删掉我手动放的封面？**

**会**。Mochi 不追踪 sidecar 来源（哪些是自己写的、哪些是你手动放的）。清除时如果勾选「包括 sidecar」，会一视同仁删除 `poster.jpg` / `fanart.jpg`。不确定就**不要勾选**。

**Q10. fonts 目录要放在哪里？**

系列文件夹**直接子目录**的 `fonts/`，与 `Season 1/` 同级。大小写不敏感。没有这个目录完全不影响播放。

**Q11. 怎么彻底退出（不留在托盘）？**

设置 → 通用 → 关闭行为 → 选「彻底退出」。或者系统托盘图标右键 → 退出。

**Q12. 数据存在哪？**

安装版：`%APPDATA%\mochi\`。便携版：exe 同级目录。`mochi.db` 是数据库，`config.json` 是配置，`cache/` 是图片缓存。

**Q13. 怎么清理缓存？**

设置 → 数据 → 图片缓存。不会删数据库和配置，图片会重新从网络拉取。

**Q14. 海报加载失败 / 显示空白？**

通常是网络问题：

1. 检查能否访问 `image.tmdb.org` 和 `lain.bgm.tv`
2. Bangumi 在国内通常需要代理
3. 尝试「刷新元数据」让系统重新拉取
4. 也可以手动放一张 `poster.jpg` 到系列文件夹下作为兜底

**Q15. 切集时 OSC 显示空轨道列表？**

v0.3.2 之前有这个问题。v0.3.3 改 `loadfile` 参数顺序时意外修了时序，v0.3.4 正式发版修复。**升级到 v0.3.4 或更新版本**。

**Q16. Bangumi 搜索没结果，但动漫明明存在？**

- 检查搜索词（中日英都试试）
- 加年份，比如「进击的巨人 2013」
- 极少数条目被 Bangumi 下架，无解

**Q17. mpv 切轨/切字幕不响应？**

v0.3.4 修复。早期版本（v0.3.2 及更早）有 `track-list` 序列化 bug。**升级**。

**Q18. 综艺在「继续」分类里没有显示？**

「继续」分类显示所有未看完剧集，包括动漫、电影、综艺。如果综艺没出现，先确认它有未看完剧集，再确认它被识别为「综艺」类型（详情页类型选择器）。

**Q19. 从 mochi 迁移到 Jellyfin / Emby 怎么操作？**

1. 设置 → 媒体库 → 批量导出 NFO
2. 关闭 mochi
3. 让 Jellyfin / Emby 扫描同一个媒体库目录
4. 元数据会被 Jellyfin / Emby 通过 NFO 读取

**Q20. macOS 什么时候支持？**

计划中。Tauri v2 跨平台理论可行，但需要单独适配 libmpv 二进制分发和一些 macOS 特有的窗口行为。

## 故障排查

### 启动崩溃 / 启动后立即关闭

1. 删除 `config.json` 重置设置（数据目录见 [数据与便携模式](#数据与便携模式)）
2. 删除 `mochi.db` 重置数据库（**会丢失所有观看进度和元数据**）
3. 重新启动

### 扫描不到视频文件

- 确认根目录添加正确
- 确认视频格式受支持（`.mkv` / `.mp4` / `.avi` / `.mov` / `.wmv` / `.flv` / `.webm` / `.ts`）
- 确认文件夹不是空目录或只有 `.nfo` / `.jpg`
- 看设置 → 媒体库 → 扫描日志

### 元数据拉到一半卡住

- 检查网络
- 检查 TMDB Key 是否过期（偶尔 TMDB 会重置）
- 在设置 → 媒体库 → 批量拉取取消后重新开始

### mpv 启动失败

- 确认 `WebView2Loader.dll` 与 exe 同级（便携版）
- 确认 `libmpv-2.dll` 与 exe 同级（便携版）
- Windows 7 / 8 用户：必须用便携版

### 字体不生效

- 确认字体在 `fonts/` 目录（不是 `Fonts.rar` 或 `fonts/` 里的子目录）
- 确认字幕里的字体名与文件名一致（`{\fn方正兰亭黑_GBK}` 对应 `方正兰亭黑_GBK.TTF`）
- 看 mpv 控制台输出（开启 verbose log）确认 `sub-fonts-dir` 被设置
